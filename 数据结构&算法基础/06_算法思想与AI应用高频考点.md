<h1 id="q-001">1. 算法思想为什么比背题更重要？</h1>

数据结构与算法面试不是单纯考记忆，而是考候选人能否把问题拆成可计算、可分析、可优化的步骤。

常见算法思想：

| 思想 | 核心问题 | 典型算法 | AI 场景 |
| --- | --- | --- | --- |
| 递归 | 问题能否拆成同类子问题 | 树遍历、DFS | 任务树、AST、计算图 |
| 分治 | 大问题拆小，结果再合并 | 归并排序、快速排序 | 分布式排序、分片聚合 |
| 减治 | 每次排除一部分 | 二分查找 | 阈值搜索、batch size 搜索 |
| 回溯 | 搜索所有候选并撤销选择 | 组合、排列、N 皇后 | 推理路径、规划搜索 |
| 动态规划 | 子问题重叠且有最优子结构 | 编辑距离、背包 | Viterbi、CTC、序列对齐 |
| 贪心 | 每步局部最优 | 区间调度、Huffman | 预算分配、启发式选择 |
| 滑动窗口 | 维护连续区间 | 最长子串、窗口最大值 | 长文本上下文、流式 token |

面试时，能说出“为什么这个问题适合某种思想”比只会背模板更重要。

<h2 id="q-002">面试问题：递归的三要素是什么？</h2>

递归三要素：

1. **递归定义**

   函数的含义是什么，输入输出是什么。

2. **终止条件**

   什么时候不再继续调用自己。

3. **递推关系**

   当前问题如何拆成更小的同类问题。

例子：二叉树最大深度。

```python
def max_depth(root):
    if not root:
        return 0
    return max(max_depth(root.left), max_depth(root.right)) + 1
```

AI 场景：

- AST 递归遍历函数、类、表达式。
- Agent 把任务递归拆成子任务。
- 计算图中递归分析依赖节点。
- JSON / YAML 配置树递归解析。

递归风险：

- 栈溢出。
- 重复计算。
- 终止条件错误导致死递归。
- 状态修改后没有恢复。

如果递归深度很大，可以改成显式栈或迭代。

<h2 id="q-003">面试问题：分治、减治和二分查找有什么关系？</h2>

分治和减治都把大问题变小，但方式不同。

| 思想 | 做法 | 例子 |
| --- | --- | --- |
| 分治 | 拆成多个子问题，分别求解，再合并 | 归并排序 |
| 减治 | 每次只保留一个子问题，丢弃其余部分 | 二分查找 |

二分查找的本质是减治：

1. 搜索区间有序。
2. 每次用中点判断目标在哪一侧。
3. 排除一半不可能区间。
4. 复杂度从 $O(n)$ 降到 $O(\log n)$。

AI 工程例子：

- 二分最大可用 batch size，找到不 OOM 的最大值。
- 二分检索阈值，平衡召回率和误召回。
- 二分采样温度或过滤阈值，满足质量指标。
- 在有序时间线上定位某次模型质量回退的版本。

面试金句：二分不只是查数组，而是在单调性存在时用判断函数快速缩小答案空间。

<h2 id="q-004">面试问题：归并排序为什么能体现分治思想？</h2>

归并排序包含两个阶段：

1. **分解**

   把数组不断拆成左右两半，直到每段长度为 1。

2. **合并**

   将两个有序数组合并成更大的有序数组。

递推式：

$$
T(n)=2T(n/2)+O(n)
$$

因此：

$$
T(n)=O(n\log n)
$$

归并排序特性：

- 稳定排序。
- 时间复杂度稳定。
- 额外空间 $O(n)$。
- 适合链表排序和外部排序。

AIGC 场景：

- 大规模训练日志可先分片排序，再多路归并。
- RAG 多路召回得到多个有序候选列表，可用归并或堆合并。
- 分布式数据处理中，map 阶段局部处理，reduce 阶段合并结果，也体现分治思想。

<h1 id="q-005">2. 回溯算法的核心思想是什么？</h1>

回溯是在搜索空间中逐步做选择，遇到不满足条件的路径就撤销选择并尝试其他路径。

适合问题：

- 组合。
- 排列。
- 子集。
- N 皇后。
- 数独。
- 路径搜索。
- 约束满足问题。

回溯的核心不是“暴力”，而是“系统性枚举 + 剪枝”。

在 AI 中，很多生成问题也像搜索：

- 多条推理路径选择。
- Agent 工具调用顺序规划。
- prompt 组合搜索。
- 超参数搜索。
- 结构化输出候选生成。

<h2 id="q-006">面试问题：回溯模板如何写？</h2>

通用模板：

```python
def backtrack(path, choices):
    if is_finished(path):
        ans.append(path[:])
        return

    for choice in choices:
        if not is_valid(path, choice):
            continue
        path.append(choice)
        backtrack(path, next_choices(choice))
        path.pop()
```

关键点：

- `path` 表示当前已做选择。
- `choices` 表示当前可选集合。
- `is_finished` 判断是否到达答案。
- `is_valid` 用于剪枝。
- `path.pop()` 撤销选择。

面试常问：

- 为什么要撤销选择？
- 什么时候需要 `used` 数组？
- 组合和排列的区别是什么？
- 如何去重重复元素？

AI 类比：

Agent 规划工具调用时，当前 path 是已执行步骤，choices 是下一步工具；如果某条路径成本过高、权限不满足或结果冲突，就需要剪枝。

<h2 id="q-007">面试问题：回溯、DFS、穷举和剪枝有什么区别？</h2>

| 概念 | 含义 |
| --- | --- |
| DFS | 一种深度优先遍历方式 |
| 穷举 | 枚举所有可能答案 |
| 回溯 | DFS 式搜索 + 做选择 + 撤销选择 |
| 剪枝 | 提前排除不可能或低价值分支 |

回溯通常使用 DFS，但 DFS 不一定是回溯。例如遍历图的连通分量是 DFS，但不一定有“选择和撤销”。

剪枝策略：

- 约束剪枝：违反条件直接跳过。
- 重复剪枝：避免重复答案。
- 边界剪枝：剩余资源不足时停止。
- 启发式剪枝：优先探索更可能成功的路径。

AIGC 场景：

- Tree-of-Thought 可以看作生成多个思路分支，然后评估和剪枝。
- Agent 多工具规划会限制最大步数、最大成本和工具权限。
- Prompt 搜索会提前丢弃低评分候选。

<h2 id="q-008">面试问题：搜索算法在 AIGC 生成和 Agent 规划中如何使用？</h2>

生成式 AI 中的搜索很常见：

| 场景 | 搜索对象 | 常用策略 |
| --- | --- | --- |
| 文本生成 | token 序列 | Greedy、Beam Search、Top K、Top P |
| 推理增强 | 思维路径 | Tree-of-Thought、Self-consistency |
| Agent 规划 | 工具调用序列 | DFS、BFS、启发式搜索 |
| 超参优化 | 学习率、batch size、采样参数 | 网格、随机、贝叶斯优化 |
| 代码修复 | patch 候选 | 生成、测试、筛选 |

Beam Search 和回溯的差异：

- 回溯可能枚举完整搜索空间。
- Beam Search 每层只保留前 B 个候选，是受限宽度搜索。

Agent 中的关键工程约束：

- 限制最大搜索深度。
- 每一步记录状态和证据。
- 对工具调用设置超时和权限。
- 对候选路径做打分。
- 避免无限循环和重复调用。

面试金句：AIGC 生成不是只靠模型一次前向，很多系统会在模型外层加搜索、评估、剪枝和验证。

<h1 id="q-009">3. 动态规划为什么是高频面试难点？</h1>

动态规划用于解决具有重叠子问题和最优子结构的问题。

两个条件：

1. **重叠子问题**

   同一子问题会被反复计算。

2. **最优子结构**

   大问题的最优解可以由子问题最优解组合得到。

两种实现：

- 自顶向下：递归 + 记忆化搜索。
- 自底向上：递推表。

DP 难点在于：

- 状态定义不清。
- 转移方程难写。
- 初始化边界容易错。
- 遍历顺序和依赖关系不匹配。
- 空间优化后可读性下降。

<h2 id="q-010">面试问题：如何设计 DP 的状态、转移和初始化？</h2>

设计 DP 可以按五步：

1. **定义状态**

   `dp[i]` 或 `dp[i][j]` 表示什么。

2. **确定转移**

   当前状态由哪些更小状态推来。

3. **初始化边界**

   空串、第一行、第一列、容量为 0 等。

4. **确定遍历顺序**

   保证转移依赖的状态已经计算。

5. **返回答案**

   答案是 `dp[n]`、`dp[n][m]`，还是表中最大值。

例子：斐波那契。

```python
def fib(n):
    if n <= 1:
        return n
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b
```

AI 场景：

- 序列对齐。
- 编辑距离。
- Viterbi 解码。
- CTC loss 的前向后向计算。
- 语音识别和 OCR 中的路径概率累积。

<h2 id="q-011">面试问题：编辑距离、LCS、LIS 分别考察什么？</h2>

| 问题 | 状态含义 | 关键思想 | AI 场景 |
| --- | --- | --- | --- |
| 编辑距离 | 两个前缀最小编辑代价 | 插入、删除、替换 | ASR/OCR 评估、文本纠错 |
| LCS | 两个前缀最长公共子序列 | 相等取斜上，不等取 max | diff、代码变更分析 |
| LIS | 以某元素结尾的最长上升子序列 | DP 或二分优化 | 排序结构、序列趋势分析 |

编辑距离转移：

如果 `a[i-1] == b[j-1]`：

$$
dp[i][j] = dp[i-1][j-1]
$$

否则：

$$
dp[i][j] = 1 + \min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
$$

分别对应删除、插入、替换。

大模型应用：

- 评估生成文本与参考答案的字符级差异。
- 代码 Agent 生成 patch 后做 diff。
- OCR 文本识别用编辑距离评估错误率。
- ASR 常用 WER，本质与编辑距离相关。

<h2 id="q-012">面试问题：DP 在语音识别、NLP 和大模型系统中有哪些应用？</h2>

DP 在 AI 中非常基础：

1. **Viterbi**

   在隐马尔可夫模型或序列标注中寻找最可能标签路径。

2. **CTC**

   语音识别、OCR 中对齐输入帧和输出标签，使用前向后向动态规划累计路径概率。

3. **编辑距离**

   用于 ASR、OCR、翻译、代码生成的评估和纠错。

4. **分词**

   传统中文分词可以用 DP 找最大概率切分路径。

5. **Diff 算法**

   代码 Agent、文档协作、版本控制中的差异比较与补丁生成。

6. **强化学习**

   值迭代和策略迭代也是动态规划思想。

注意：Transformer 本身主要依赖矩阵计算和注意力机制，但大模型系统周边仍大量使用 DP 做对齐、评估、解码和工具链处理。

<h1 id="q-013">4. 贪心、双指针和滑动窗口解决什么问题？</h1>

贪心、双指针、滑动窗口都是降低搜索成本的高频技巧。

| 技巧 | 核心 | 常见问题 |
| --- | --- | --- |
| 贪心 | 每一步做局部最优选择 | 区间调度、最少跳跃、Huffman |
| 双指针 | 两个指针协同移动 | 有序数组、链表、快慢指针 |
| 滑动窗口 | 维护连续区间状态 | 子串、子数组、流式窗口 |

AIGC 场景：

- 长文本截断和滑窗摘要。
- RAG chunk 合并和窗口检索。
- 流式输出 token 缓冲。
- GPU 请求动态 batching。
- 多模态视频帧采样和片段窗口。

<h2 id="q-014">面试问题：贪心算法什么时候可靠？</h2>

贪心每一步选择当前看起来最优的方案。它可靠通常需要：

1. **贪心选择性质**

   局部最优选择可以导向全局最优。

2. **最优子结构**

   做出选择后，剩余问题仍然是同类最优化问题。

常见例子：

- 区间调度选择最早结束的活动。
- Huffman 编码每次合并频率最小的两个节点。
- Dijkstra 每次确定当前最近节点，要求边权非负。

AI 工程中的贪心更多是启发式：

- 在 token budget 不够时优先保留最近上下文。
- RAG 中优先保留 rerank 分数最高的 chunk。
- 调度系统优先执行短任务或高优先级任务。
- Agent 优先选择成本低、成功率高的工具。

这些启发式不一定全局最优，但常在工程上足够有效。面试中要区分“可证明贪心”和“工程启发式贪心”。

<h2 id="q-015">面试问题：双指针和滑动窗口有什么区别？</h2>

双指针是更宽泛的技巧，滑动窗口是双指针的一种。

双指针常见类型：

- 左右指针：用于有序数组两数之和、反转字符串。
- 快慢指针：用于链表中点、环检测。
- 同向指针：用于合并两个有序数组。

滑动窗口：

- 通常维护连续区间 `[left, right]`。
- 右指针扩展窗口。
- 左指针收缩窗口。
- 同时维护窗口内统计量。

例子：最长无重复子串。

```python
def length_of_longest_substring(s):
    seen = {}
    left = 0
    ans = 0
    for right, ch in enumerate(s):
        if ch in seen and seen[ch] >= left:
            left = seen[ch] + 1
        seen[ch] = right
        ans = max(ans, right - left + 1)
    return ans
```

AI 场景：

- 滑动窗口切分长文档。
- 视频模型按窗口处理帧序列。
- LLM 流式输出中维护最近 token。
- 监控系统统计最近 5 分钟错误率。

<h2 id="q-016">面试问题：长上下文、chunking 和 token budget 如何用窗口思想理解？</h2>

长上下文处理本质上常是窗口管理问题。

典型问题：

- 模型上下文长度有限。
- 文档过长不能一次放入 prompt。
- 多轮对话历史不断增长。
- RAG 返回 chunk 过多会挤占生成空间。

窗口思想：

1. **固定窗口**

   按固定 token 数切分文档，简单但可能切断语义。

2. **重叠窗口**

   相邻 chunk 保留 overlap，减少边界信息丢失。

3. **动态窗口**

   根据标题、段落、语义边界自适应切分。

4. **滑动上下文**

   推理时保留最近对话或最近工具结果。

5. **预算分配**

   prompt 中系统指令、用户问题、检索证据和输出空间需要共同分配 token budget。

工程建议：

- chunk 不是越大越好，过大影响召回精度。
- overlap 不是越多越好，过多会增加重复和成本。
- 长上下文不等于不需要检索，注意力和成本仍有限。
- 对多轮对话可结合摘要、记忆和滑动窗口。

<h1 id="q-017">5. 为什么 AI 算法岗也会考数值算法和综合设计？</h1>

AI 算法岗不仅写模型，也需要理解数值稳定性、线性方程、优化迭代和工程复杂度。

常见数值与综合问题：

- 牛顿迭代。
- 高斯消元。
- 浮点误差。
- 矩阵求解。
- Top K 与排序优化。
- 大规模去重和检索。
- RAG 系统设计。
- LLM 推理缓存和调度优化。

这些题考察候选人是否能把数学、算法和工程系统联系起来。

<h2 id="q-018">面试问题：如何用牛顿迭代求平方根？</h2>

要求 $\sqrt{x}$，可转化为求方程：

$$
f(t)=t^2-x=0
$$

牛顿迭代公式：

$$
t_{k+1}=t_k-\frac{f(t_k)}{f'(t_k)}
$$

因为 $f'(t)=2t$，所以：

$$
t_{k+1}=\frac{1}{2}\left(t_k+\frac{x}{t_k}\right)
$$

代码示例：

```python
def sqrt_newton(x, eps=1e-8):
    if x < 0:
        raise ValueError("x must be non-negative")
    if x == 0:
        return 0
    t = x
    while abs(t * t - x) > eps:
        t = 0.5 * (t + x / t)
    return t
```

AI 相关理解：

- 牛顿法是优化算法的重要基础。
- 二阶优化利用梯度和曲率信息。
- 深度学习常用一阶优化，但数值迭代思想仍非常重要。
- 实际工程要关注初值、收敛条件、除零和浮点误差。

<h2 id="q-019">面试问题：高斯消元如何求解线性方程组？</h2>

高斯消元用于求解线性方程组：

$$
A\mathbf{x}=\mathbf{b}
$$

核心思想是通过初等行变换将增广矩阵化为上三角矩阵，再回代求解。

流程：

1. 选择主元。
2. 用主元所在行消去下面各行对应列。
3. 重复直到形成上三角矩阵。
4. 从最后一行开始回代。

复杂度通常为 $O(n^3)$。

工程注意：

- 主元接近 0 时会导致数值不稳定。
- 常使用部分主元选择提高稳定性。
- 稀疏矩阵应使用专门的稀疏求解器。
- 大规模深度学习不会直接频繁做普通高斯消元，但线性代数求解仍是科学计算基础。

AI 场景：

- 最小二乘问题。
- SLAM 和 bundle adjustment 中的线性化求解。
- 传统机器学习中的线性模型。
- 图优化和几何视觉中的方程求解。

<h2 id="q-020">面试问题：如何用算法思维优化 RAG、训练数据处理和 LLM 推理？</h2>

可以从三个系统分别回答。

**1. RAG 优化**

- 用倒排索引处理精确关键词召回。
- 用向量索引做语义召回。
- 用哈希表维护 doc_id 到元数据映射。
- 用 Top K 堆合并多路候选。
- 用 reranker 排序提升证据质量。
- 用布隆过滤器或 bitmap 做去重和过滤。
- 用缓存减少重复 query 的 embedding 和检索开销。

**2. 训练数据处理优化**

- 用哈希和布隆过滤器做样本去重。
- 用桶排序思想按长度分桶，减少 padding 浪费。
- 用队列和预取减少 GPU 等待。
- 用并行 map-reduce 思想处理超大数据集。
- 用滑动窗口切分长文本。
- 用统计索引追踪数据来源、版本和质量。

**3. LLM 推理优化**

- KV Cache 用空间换时间。
- Prefix cache 用 Trie / Radix tree 复用公共前缀。
- Continuous batching 用队列和调度算法提高吞吐。
- Top K / Top P 避免完整排序。
- Speculative decoding 用草稿模型减少目标模型调用。
- 请求调度需要平衡吞吐、延迟和公平性。

面试金句：

AI 系统性能优化往往不是单点模型优化，而是数据结构、算法复杂度、缓存、索引、调度和硬件执行共同优化。

## 高频速记

- 递归看定义、终止条件、递推关系。
- 分治拆多个子问题并合并，减治每次排除一部分。
- 二分查找本质是利用单调性缩小答案空间。
- 回溯 = 做选择、递归、撤销选择，剪枝决定效率。
- Beam Search 是受限宽度搜索，不是完整穷举。
- DP 适合重叠子问题和最优子结构。
- 编辑距离、LCS、Viterbi、CTC 都体现 DP 思想。
- 贪心要区分可证明正确和工程启发式。
- 滑动窗口适合连续区间和流式数据。
- 牛顿迭代体现数值优化思想，高斯消元体现线性代数求解思想。
