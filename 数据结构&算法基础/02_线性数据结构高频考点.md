<h1 id="q-001">1. 数组为什么是最基础的数据结构？</h1>

数组使用连续内存存储一组同类型元素，支持通过下标进行 $O(1)$ 随机访问。

数组的核心优势：

- 内存连续，缓存友好。
- 按下标访问快。
- 适合批量计算和向量化。
- 结构简单，便于序列化。

数组的局限：

- 插入和删除中间元素需要搬移，时间复杂度 $O(n)$。
- 固定数组容量不易扩展。
- 需要连续内存，超大数组可能分配失败。

AI 中的数组例子：

- 图像张量 `[B, C, H, W]`。
- 文本 token id 序列。
- embedding table。
- batch 内样本列表。
- attention mask。
- logits 数组。

<h2 id="q-002">面试问题：数组和动态数组有什么区别？</h2>

固定数组容量在创建时确定；动态数组可以在容量不足时扩容。

动态数组扩容过程：

1. 申请更大连续空间。
2. 将旧元素复制到新空间。
3. 释放旧空间。
4. 新元素写入新空间。

复杂度：

- 按下标访问：$O(1)$。
- 尾部追加：摊还 $O(1)$。
- 中间插入/删除：$O(n)$。
- 扩容单次：$O(n)$。

为什么 append 是摊还 $O(1)$？

扩容不是每次都发生。假设容量每次翻倍，虽然某次扩容要复制很多元素，但均摊到多次 append 后，平均成本仍是常数级。

AI 场景：

- 收集一个 batch 的样本可以用动态数组。
- 保存变长 token 序列时，通常先用动态容器收集，再 padding 成 Tensor。
- 日志和 trace 事件可先 append 到列表，结束后批量写入。

<h2 id="q-003">面试问题：字符串为什么可以看作特殊数组？</h2>

字符串本质上是字符序列，可以理解为字符数组。

区别：

- 字符串通常是不可变或半不可变结构。
- C/C++ 字符串常以 `\0` 结尾。
- Python 字符串不可变，频繁拼接会产生新对象。
- 字符串算法更关注匹配、前缀、后缀、编辑距离和编码。

AIGC 场景：

- Prompt 是字符串。
- Tokenizer 将字符串转换为 token id 数组。
- 敏感词过滤可用 Trie / AC 自动机。
- LLM 输出流是字符串片段序列。
- 代码 Agent 需要对源码字符串做 diff、patch、解析。

面试中要补充：大模型不直接处理原始字符串，而是先经过 tokenizer 变成整数 token 序列，再进入 embedding。

<h2 id="q-004">面试问题：Tensor 与数组有什么关系？</h2>

Tensor 可以理解为多维数组，是深度学习中的核心数据结构。

| 概念 | 数组 | Tensor |
| --- | --- | --- |
| 维度 | 通常一维或多维 | 任意维 |
| 数据类型 | 同类型元素 | dtype 明确，如 fp32、bf16、int64 |
| 内存布局 | 连续或按语言实现 | stride、contiguous、device |
| 运算 | 普通循环 | 向量化、GPU kernel、自动求导 |

Tensor 的关键属性：

- shape。
- dtype。
- device。
- stride。
- contiguous。
- requires_grad。

AI 工程中常见坑：

- `view` 要求内存连续，`reshape` 可能触发拷贝。
- transpose 后 Tensor stride 变化，可能影响性能。
- CPU/GPU 之间拷贝很贵。
- dtype 不匹配会导致额外转换或精度问题。

<h1 id="q-005">2. 链表的核心思想是什么？</h1>

链表通过指针或引用将节点串联起来。每个节点通常包含数据和指向下一个节点的指针。

常见类型：

- 单链表。
- 双向链表。
- 循环链表。

优势：

- 插入和删除节点只需修改指针，已知节点时为 $O(1)$。
- 不要求连续内存。
- 适合频繁移动节点的场景。

局限：

- 不支持 $O(1)$ 随机访问。
- 需要额外指针空间。
- 缓存局部性差。
- 代码更容易出现空指针、断链和循环错误。

AI 工程中链表不如数组常见，但在 LRU、内存块管理、调度队列等场景仍有价值。

<h2 id="q-006">面试问题：数组和链表如何对比？</h2>

| 维度 | 数组 | 链表 |
| --- | --- | --- |
| 内存 | 连续 | 不连续 |
| 随机访问 | $O(1)$ | $O(n)$ |
| 插入删除 | 中间位置 $O(n)$ | 已知节点 $O(1)$ |
| 缓存友好 | 好 | 差 |
| 额外空间 | 少 | 指针开销 |
| 实现复杂度 | 简单 | 较复杂 |

AI 场景中，绝大多数数值计算优先数组/Tensor，因为连续内存和向量化更重要。链表适合节点频繁移动且不需要随机访问的场景，例如 LRU 双向链表。

<h2 id="q-007">面试问题：快慢指针为什么能求链表中点和判断环？</h2>

快慢指针是链表高频技巧：

- 慢指针每次走 1 步。
- 快指针每次走 2 步。

求中点：

```python
def middle_node(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow
```

判断环：

- 如果链表有环，快指针最终会追上慢指针。
- 如果无环，快指针会先到 `None`。

应用：

- 链表中点。
- 环检测。
- 删除倒数第 k 个节点。
- 判断回文链表。

AI 工程类比：

- 双指针思想常用于滑动窗口、token 序列截断、流式缓冲区处理。
- 快慢指针本质是用不同速度扫描来发现结构性质。

<h2 id="q-008">面试问题：如何用哈希表 + 双向链表实现 LRU Cache？</h2>

LRU Cache 要支持：

- `get(key)`：查找并把节点移到最近使用位置。
- `put(key, value)`：插入或更新，并在超容量时淘汰最久未使用节点。

单独链表的问题：查找慢，$O(n)$。

单独哈希表的问题：不知道谁最久未使用。

组合方案：

```text
HashMap: key -> Node
DoubleLinkedList: 按访问时间维护节点顺序
```

复杂度：

- 查询：$O(1)$。
- 插入：$O(1)$。
- 删除：$O(1)$。
- 移动到头部：$O(1)$。

AI 场景：

- RAG query cache。
- embedding cache。
- 多轮对话 session cache。
- 推理服务中的 prefix / KV cache 淘汰。
- 特征工程中热门特征缓存。

面试中可以强调：LRU 是数据结构组合题，不是单一结构题；哈希表负责定位，双向链表负责顺序。

<h1 id="q-009">3. 栈和队列分别解决什么问题？</h1>

栈和队列都是线性结构，但访问规则不同：

- 栈：后进先出，LIFO。
- 队列：先进先出，FIFO。

栈适合处理嵌套、回退、递归和最近状态。

队列适合处理排队、层序、任务调度和流式数据。

<h2 id="q-010">面试问题：栈为什么和递归、DFS、表达式解析有关？</h2>

函数调用本身使用调用栈。每次递归调用都会把局部变量、返回地址等压入栈。

栈的典型应用：

- DFS。
- 括号匹配。
- 表达式求值。
- 单调栈。
- 浏览器前进后退。
- 递归转迭代。

AI 场景：

- 解析模型配置或表达式。
- 遍历 AST。
- 代码 Agent 分析嵌套函数。
- 深度优先搜索解码或规划。
- 回溯搜索中的路径维护。

注意：

- 递归深度过大可能栈溢出。
- Python 中递归层数有限，面试和工程中可改成显式栈。

<h2 id="q-011">面试问题：队列为什么和 BFS、任务调度、DataLoader 有关？</h2>

队列按先来先处理组织数据，适合层序扩展和生产者-消费者模型。

典型应用：

- BFS。
- 任务队列。
- 消息队列。
- 线程池任务分发。
- DataLoader 预取。
- 流式输出缓存。

AI 场景：

- DataLoader 多 worker 将 batch 放入队列，训练进程消费。
- 推理服务将请求放入队列，调度器合并成 dynamic batch。
- Agent 后台任务通过队列异步执行。
- BFS 可用于知识图谱邻居扩展、网页爬取、workflow 遍历。

面试中可以说：队列是 AI 系统吞吐优化的重要结构，能解耦生产速度和消费速度。

<h2 id="q-012">面试问题：循环队列和阻塞队列在 AI 工程中有什么用？</h2>

**循环队列** 用固定数组实现队列，使用 head / tail 指针环形移动。

优点：

- 避免频繁搬移元素。
- 空间固定。
- 适合高吞吐缓冲区。

典型场景：

- 流式 token buffer。
- 日志环形缓冲。
- 实时音视频帧缓存。
- 传感器数据窗口。

**阻塞队列** 用于生产者-消费者：

- 队列空时消费者等待。
- 队列满时生产者等待。

AI 场景：

- 数据读取线程生产 batch，训练线程消费 batch。
- Web 服务接收请求，推理 worker 消费请求。
- 多模态系统中音频帧、视频帧、文本事件异步传递。

<h1 id="q-013">4. 单调栈和单调队列解决什么问题？</h1>

单调结构用于在 $O(n)$ 时间内处理“最近更大/更小元素”“滑动窗口最值”等问题。

核心思想：

- 栈或队列中元素保持单调。
- 新元素进入时，弹出不可能再成为答案的旧元素。
- 每个元素最多入栈/入队一次、出栈/出队一次，所以总复杂度 $O(n)$。

<h2 id="q-014">面试问题：单调栈如何在 $O(n)$ 内找到下一个更大元素？</h2>

问题：对数组中每个元素，找到右边第一个比它大的元素。

模板：

```python
def next_greater(nums):
    ans = [-1] * len(nums)
    stack = []  # 存下标，保持对应值单调递减
    for i, x in enumerate(nums):
        while stack and nums[stack[-1]] < x:
            ans[stack.pop()] = x
        stack.append(i)
    return ans
```

为什么是 $O(n)$？

- 每个元素最多入栈一次。
- 每个元素最多出栈一次。

AI 场景类比：

- 在时序特征中找未来第一个超过阈值的点。
- 在显存使用曲线中找下一次峰值。
- 在日志序列中找下一次更高严重级别事件。

<h2 id="q-015">面试问题：单调队列如何优化滑动窗口最大值？</h2>

问题：给定数组和窗口大小 $k$，求每个窗口最大值。

模板：

```python
from collections import deque

def max_sliding_window(nums, k):
    q = deque()  # 存下标，对应值单调递减
    ans = []
    for i, x in enumerate(nums):
        while q and q[0] <= i - k:
            q.popleft()
        while q and nums[q[-1]] <= x:
            q.pop()
        q.append(i)
        if i >= k - 1:
            ans.append(nums[q[0]])
    return ans
```

AI 场景：

- 时间序列特征的滑动窗口最大值。
- 流式监控中的 P99 / 最大延迟近似维护。
- 音频、视频信号中的局部峰值检测。
- 训练 loss 或 GPU utilization 的窗口统计。

<h2 id="q-016">面试问题：线性结构在流式大模型推理中有哪些应用？</h2>

流式大模型推理中，线性结构非常常见：

| 结构 | 应用 |
| --- | --- |
| 动态数组 | 收集 token、logits、trace event |
| 队列 | 请求排队、dynamic batching、流式输出 |
| 环形缓冲 | 固定长度上下文窗口、日志窗口 |
| 栈 | 解析工具调用、嵌套 JSON、代码块 |
| 双端队列 | 滑动窗口、流式 token 截断 |
| LRU | prefix cache、RAG cache、session cache |

例子：聊天模型流式输出时，服务端通常一边生成 token，一边将 token 放入输出队列，由网络层消费并推送给前端。如果消费者慢，队列还要考虑背压。

## 高频速记

1. 数组连续内存、随机访问快，是 Tensor 的基础。
2. 动态数组 append 摊还 $O(1)$，中间插删 $O(n)$。
3. 链表插删灵活但缓存不友好，常用于 LRU 的双向链表。
4. LRU = 哈希表定位 + 双向链表维护访问顺序。
5. 栈适合递归、DFS、表达式解析、单调栈。
6. 队列适合 BFS、任务调度、DataLoader、消息系统。
7. 循环队列适合固定容量流式缓冲。
8. 单调栈解决下一个更大/更小元素，单调队列解决滑动窗口最值。
9. 流式 LLM 推理大量使用队列、环形缓冲、LRU 和 token 数组。
