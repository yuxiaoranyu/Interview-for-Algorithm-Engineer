## 后续补充格式建议

每个公司文件建议按照以下结构持续更新：

1. 岗位与轮次：例如“大模型算法岗一面”“推荐算法岗二面”“多模态算法实习岗”。
2. 原始问题：尽量保留面试官原问法，便于检索。
3. 面试答案：给出可直接口述的核心答案、公式、工程经验和常见追问。
4. 手撕代码：代码题单独保留复杂度、边界情况和可运行实现。
5. 复盘要点：补充该公司业务场景、考察重点和准备建议。
