# 配图统一风格要求

所有图片统一采用同一套视觉风格：白色背景，蓝灰色为主色，青蓝色作为辅助色，橙色仅用于强调风险、瓶颈、对比或长尾问题；整体为简洁专业的扁平化技术信息图风格，适合算法工程师面试文档和 Markdown 技术文档配图。

统一规范：
- 图片比例统一为 16:9。
- 图片优先按高清规格生成，建议至少 1920×1080；不要依赖生成大图后缩放来解决文字清晰度问题，文字应在最终展示尺寸下仍清晰可读。
- 中文字体使用宋体，英文字体使用 Times New Roman，字号统一为 Word 规范中的四号（14 pt）。
- 所有文字必须清晰锐利、无模糊、无重影、无变形，适合在 Markdown 或 Word 文档中直接阅读。
- 尽量避免密集文本，不使用大段说明文字；复杂解释放到正文中，图中只保留关键词和核心关系。
- 使用统一的圆角矩形卡片、细线箭头、浅色填充、低饱和描边和少量图标化元素。
- 所有模块间距、边距、箭头粗细、卡片圆角、标题层级保持一致。
- 不使用真实公司 logo，不使用人物、照片、复杂背景或 3D 效果。
- 图中信息优先保证清晰可读，避免装饰性元素过多。

## 推荐生成流程：两阶段生成

如果直接生成带文字图片时出现文字模糊、重影、变形、乱码或字号不稳定，优先采用"两阶段生成"：

**第一阶段：生成无文字底图**
- 只生成布局、卡片、箭头、图标、颜色、模块结构和留白。
- 卡片内不写正式文字，只保留空白区域或极少量占位符。
- 明确要求不要生成小字、长句、乱码文字和装饰性伪文字。
- 输出 16:9 高清底图，至少 1920×1080。

**第二阶段：添加文字层**
- 在第一阶段底图基础上，只添加或重绘文字层。
- 不改变布局、结构、配色、图标、卡片位置、箭头和信息层级。
- 中文使用宋体，英文使用 Times New Roman，字号不小于 14 pt。
- 每个卡片最多保留标题 + 2-4 个关键词，文字居中或左对齐良好，不能贴边。
- 所有文字必须清晰锐利、无模糊、无重影、无扭曲、无乱码。

两阶段生成的目标是：**让图像模型先稳定生成结构和视觉风格，再单独控制文字排版和清晰度**。如果追求最高可控性，也可以用第一阶段生成的无文字底图，在 PPT、Figma、draw.io 或 Word 中手工添加文字。
