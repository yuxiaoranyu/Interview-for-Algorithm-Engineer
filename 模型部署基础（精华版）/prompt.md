# 配图统一风格要求

所有图片统一采用同一套视觉风格：白色背景，蓝灰色为主色，青蓝色作为辅助色，橙色仅用于强调风险、瓶颈、对比或长尾问题；整体为简洁专业的扁平化技术信息图风格，适合算法工程师面试文档和 Markdown 技术文档配图。

统一规范：
- 图片比例统一为 16:9。
- 图片优先按高清规格生成，建议至少 1920×1080。
- 中文字体使用 "SimSun (宋体) font"，英文字体使用 "Times New Roman font"，字号统一为 Word 规范中的四号（14 pt）。
- 所有文字必须清晰锐利、无模糊、无重影、无变形，适合在 Markdown 或 Word 文档中直接阅读。
- 每个卡片最多保留标题 + 2-4 个关键词，单张图中文总量控制在 30 字以内；复杂解释放到正文中。
- 使用统一的圆角矩形卡片、细线箭头、浅色填充、低饱和描边和少量图标化元素。
- 所有模块间距、边距、箭头粗细、卡片圆角、标题层级保持一致。
- 不使用真实公司 logo，不使用人物、照片、复杂背景或 3D 效果。
- 图中信息优先保证清晰可读，避免装饰性元素过多。

## GPT Image 2 文字渲染专项要求

基于 GPT Image 2 最佳实践，所有提示词须遵守：
- **引号包裹**：所有需要在图中显示的中英文文字用双引号 `"..."` 单独标出，帮助模型区分"要写的字"和"描述性指令"。
- **直接写出完整文字**：不写"放上标题和副标题"，而是直接写出具体文字内容、字体和位置。
- **逐行定位**：多行文字按"第一行/第二行"明确排版结构和对齐方式。
- **预留留白**：文字不贴卡片边框，四周保留充足留白；文字不与箭头、图标重叠。
- **分散排布**：文字分布在模块两侧留白区、四角留白区、边缘排版区，不粗暴压在模块中段中心。
