<h1 id="q-001">1. 为什么 AIGC 时代仍然需要 3D 数字人？</h1>

AIGC 视频模型可以直接生成高保真人像视频，但 3D 数字人仍然非常重要，因为它提供了 2D 生成模型不容易稳定保证的能力：**几何一致性、视角一致性、姿态可控、实时驱动、可编辑资产和工程可部署性**。

在数字人系统中，3D 不是为了替代 2D 生成，而是作为一个强先验：

- **几何先验**：脸型、头部姿态、人体骨架、深度、法线、遮挡关系更稳定。
- **运动先验**：身体骨架、面部 BlendShape、手部关节可以被动作捕捉或音频驱动。
- **渲染先验**：相机、光照、材质、阴影可以被显式控制。
- **资产先验**：Mesh、贴图、Rig、骨骼动画可以进入 Unity、Unreal、WebGL、AR/VR 等实时引擎。
- **生成约束**：ControlNet、Adapter、视频扩散模型可以用深度图、法线图、UV、DensePose、SMPL 渲染图作为条件，减少身份漂移和肢体错误。

**一句话总结：**

2D 生成擅长真实感和内容多样性，3D 数字人擅长可控性、可解释性和跨视角一致性。AIGC 时代的主流工程路线通常是二者融合。

<h2 id="q-002">面试问题：2D 数字人与 3D 数字人的核心差异是什么？</h2>

| 维度 | 2D 数字人 | 3D 数字人 |
| --- | --- | --- |
| 表示方式 | 图像、视频、特征图、隐空间 | Mesh、点云、体素、SDF、NeRF、3DGS、骨骼、材质 |
| 生成优势 | 真实感强、训练数据多、风格迁移灵活 | 姿态可控、视角一致、可实时驱动、资产可复用 |
| 主要难点 | 时序闪烁、身份漂移、手部/遮挡错误 | 建模成本高、纹理/头发/衣物复杂、真实感依赖渲染 |
| 适合任务 | 口播视频、换脸、换衣、短视频生成 | 虚拟主播、游戏角色、AR/VR、全身交互、动作捕捉 |
| 工程依赖 | 扩散模型、视频生成、检测分割、人脸识别 | 图形学、Rig、渲染引擎、可微渲染、3D 重建 |

面试中不要把 2D 和 3D 讲成替代关系。更准确的回答是：

- **2D 负责观感上限**：让图像更像真实视频。
- **3D 负责控制下限**：确保动作、视角、结构、遮挡不崩。
- **混合式系统最常见**：先用 3D 或几何条件控制结构，再用生成模型补纹理、细节和真实感。

<h2 id="q-003">面试问题：3D 数字人的典型技术链路是什么？</h2>

一个典型 3D 数字人系统通常包含以下模块：

1. **资产构建**

   通过扫描、多视角重建、单图重建、参数化模型拟合或生成式 3D 方法得到人脸/人体几何、纹理、头发、衣物和材质。

2. **参数化表示**

   人脸常用 3DMM、FLAME、BlendShape；人体常用 SMPL、SMPL-X、MANO；更高保真的 Avatar 会进一步叠加衣物网格、头发表示、动态纹理和神经渲染模块。

3. **驱动控制**

   输入可以是文本、音频、视频、动作捕捉、关键点、骨骼序列或表情参数。系统将这些输入转换为姿态、表情、口型、手势和相机轨迹。

4. **渲染生成**

   传统路线用 PBR/实时渲染引擎输出画面；神经路线用 NeRF、3D Gaussian Splatting 或神经纹理增强；混合路线把 3D 渲染结果作为扩散模型控制条件。

5. **后处理与上线**

   包括人脸增强、超分、时序一致性、口型校准、背景融合、绿幕/透明通道、模型压缩、流式推理和端侧部署。

<h2 id="q-004">面试问题：3D 先验如何提升数字人生成的可控性？</h2>

3D 先验的价值在于把难以直接从像素中稳定学习的结构变量显式化。

常见方式包括：

- **用 3DMM/FLAME 控制脸部**：将头部姿态、表情、眼球、下颌、嘴型转为可渲染条件。
- **用 SMPL/SMPL-X 控制身体**：把人体姿态、体型、手势和面部表情变成统一参数。
- **用深度/法线/分割控制生成**：给扩散模型提供空间约束，减少肢体错位、脸部变形和服装漂移。
- **用 UV/纹理空间保持身份**：把人脸或身体纹理绑定到稳定拓扑，跨帧复用。
- **用可微渲染做反向优化**：通过图像损失反推相机、姿态、形状、表情、光照和材质参数。

例如人像视频生成可以先估计每帧 FLAME 参数，再渲染法线图、深度图和语义图，最后用视频扩散模型生成高保真帧。这样模型不需要凭空“猜”头部旋转和嘴型结构。

<h1 id="q-005">2. 3DMM 是什么？</h1>

3DMM，全称 3D Morphable Model，是一种基于统计形变空间的三维人脸模型。它把人脸几何和纹理表示为平均脸与若干主成分基的线性组合，从而用低维参数生成不同身份、表情和纹理的人脸。

经典形式可以写成：

$$
S = \bar{S} + A_{\text{id}}\alpha + A_{\text{exp}}\beta
$$

$$
T = \bar{T} + A_t\gamma
$$

其中：

- $S$ 表示三维人脸形状。
- $\bar{S}$ 表示平均脸形状。
- $A_{\text{id}}$ 表示身份形状基。
- $\alpha$ 表示身份参数。
- $A_{\text{exp}}$ 表示表情基。
- $\beta$ 表示表情参数。
- $T$ 表示纹理或反照率。
- $A_t$ 和 $\gamma$ 分别表示纹理基与纹理参数。

3DMM 的核心思想是：虽然真实人脸空间很复杂，但大部分常见人脸形状可以落在一个低维统计子空间中。因此从单张图像估计少量参数，就能恢复一个可解释、可渲染、可动画的人脸模型。

<h2 id="q-006">面试问题：3DMM 的形状、表情和纹理参数如何理解？</h2>

3DMM 中常见参数可以分为三类：

| 参数 | 含义 | 主要控制内容 |
| --- | --- | --- |
| 身份形状参数 | identity shape | 脸宽、脸长、颧骨、下巴、鼻梁、眼距等相对稳定特征 |
| 表情参数 | expression | 张嘴、微笑、皱眉、眯眼、脸颊运动等动态变形 |
| 纹理参数 | texture/albedo | 肤色、眉毛、嘴唇、局部颜色分布等外观特征 |

还会配合估计：

- **相机参数**：旋转、平移、尺度、焦距或弱透视投影。
- **光照参数**：常用球谐光照近似环境光。
- **姿态参数**：头部 yaw、pitch、roll。
- **局部细节参数**：皱纹、法线、displacement 或 detail code。

面试回答时要强调：身份参数不应该包含表情，纹理参数不应该把光照误当成肤色。实际训练中经常需要通过正则项、光照建模和多视角/多帧约束来减少参数耦合。

<h2 id="q-007">面试问题：3DMM 有哪些典型应用？</h2>

3DMM 在数字人中常见应用包括：

1. **单图/视频人脸重建**

   从图像中估计三维脸型、头部姿态、表情和纹理，用于建模或后续编辑。

2. **表情驱动动画**

   将音频、视频或表情捕捉结果映射到 3DMM 表情参数，再驱动数字人说话、眨眼、微笑和转头。

3. **换脸与身份保持**

   3DMM 可以提供脸部几何对齐、姿态归一化、mask、法线和 UV 映射，帮助处理大姿态、遮挡和光照差异。

4. **人脸编辑与美化**

   通过调整形状、表情、纹理和光照参数实现瘦脸、表情编辑、视角重定向、表情迁移等。

5. **可微优化**

   将 3DMM 渲染到图像平面后，用 landmark loss、photometric loss、identity loss 等反向优化参数。

6. **生成模型控制条件**

   渲染出的 depth、normal、landmark、semantic map 可以作为扩散模型的结构条件，提升脸部稳定性。

<h2 id="q-008">面试问题：BFM 和 FLAME 有什么区别？</h2>

BFM 和 FLAME 都属于三维人脸/头部统计模型，但建模范围和目标不同。

| 维度 | BFM | FLAME |
| --- | --- | --- |
| 全称 | Basel Face Model | Faces Learned with an Articulated Model and Expressions |
| 建模对象 | 主要是人脸区域 | 完整头部，包含脸、头颅、脖子等 |
| 重点 | 身份形状和纹理统计空间 | 头部姿态、脖子、眼球、下颌和表情动画 |
| 动画能力 | 经典 BFM 本身偏静态 | 更适合表情和头部动画 |
| 纹理 | 通常包含纹理空间 | 原始 FLAME 更偏几何，可与纹理模型结合 |
| 数字人应用 | 人脸重建、身份拟合、纹理建模 | Talking head、表情驱动、头部 avatar、可控视频生成 |

简洁回答：

BFM 更像一个经典的人脸形状/纹理统计模型；FLAME 更像一个适合动画驱动的完整头部参数化模型，显式考虑脖子、下颌、眼球和表情。

<h2 id="q-009">面试问题：DECA、EMOCA、MICA、3DDFA、Deep3DFaceRecon 的路线如何区分？</h2>

这些方法都服务于 3D 人脸重建，但目标侧重点不同。

| 方法 | 主要特点 | 适合理解成 |
| --- | --- | --- |
| 3DDFA/3DDFA-V2 | 快速密集人脸对齐，偏实时鲁棒 | 工程友好的人脸 3D 对齐方法 |
| Deep3DFaceRecon | 基于 3DMM 的深度人脸重建，常用于身份、姿态、光照估计 | 经典 3DMM 回归路线 |
| DECA | 基于 FLAME，强调细节表情捕捉和可动画细节 | 高质量人脸细节重建 |
| EMOCA | 在 DECA 基础上更重视情绪和表情语义 | 表情感知的人脸重建 |
| MICA | 强调身份级高精度几何，常与 FLAME 结合 | 更稳定的身份形状估计 |

面试中可以这样概括：

- **3DDFA** 偏快速对齐和稳定姿态。
- **Deep3DFaceRecon** 偏标准 3DMM 参数估计。
- **DECA/EMOCA** 偏细节、表情和可动画 Avatar。
- **MICA** 偏身份形状的准确性和跨图一致性。

真实系统经常组合使用：先做人脸检测和关键点，再估计 3DMM/FLAME 参数，最后用神经渲染或扩散模型增强纹理细节。

<h2 id="q-010">面试问题：单张图像做人脸 3D 重建的难点有哪些？</h2>

单图 3D 人脸重建本质上是从二维投影恢复三维结构，是一个病态逆问题。主要难点包括：

1. **深度歧义**

   同一张 2D 图可能对应多个 3D 脸型、相机和光照组合。

2. **姿态和遮挡**

   大侧脸、手部遮挡、头发、眼镜、口罩会导致关键区域不可见。

3. **光照与纹理耦合**

   图像颜色同时受肤色、光照、阴影、相机曝光影响，很难直接分解出真实 albedo。

4. **表情与身份耦合**

   张嘴、微笑、皱眉会改变几何，如果没有分离好，模型可能把临时表情误认为身份脸型。

5. **高频细节缺失**

   低维 3DMM 难以表达皱纹、毛孔、胡须、皮肤细节和头发边界。

6. **数据和监督不足**

   高质量 3D 扫描数据昂贵，真实场景中标注的 3D ground truth 很少。

常见缓解策略：

- 使用 landmark loss、photometric loss、identity loss、regularization loss 联合训练。
- 引入多帧、多视角或视频时序约束。
- 使用 FLAME/3DMM 作为低频几何，再预测 detail normal/displacement。
- 对遮挡区域使用 segmentation、visibility mask 或鲁棒损失。

<h1 id="q-011">3. SMPL 模型是什么？</h1>

SMPL，全称 Skinned Multi-Person Linear Model，是一种参数化三维人体模型。它用低维形状参数和姿态参数生成统一拓扑的人体 Mesh，并且兼容传统图形学中的骨骼蒙皮管线。

SMPL 的典型输入包括：

- **形状参数 $\beta$**：控制身高、胖瘦、肩宽、腿长等体型差异。
- **姿态参数 $\theta$**：控制身体各关节旋转。
- **全局位姿**：控制人体整体朝向和平移。

SMPL 的典型输出包括：

- 约 6890 个顶点的人体 Mesh。
- 约 24 个身体关节。
- 与姿态和体型一致的三维人体表面。

它的关键价值是：用很少的参数表示可动画、可渲染、可优化的人体几何。因此 SMPL 成为动作捕捉、人体重建、虚拟试衣、人体生成、姿态估计和数字人动画中的基础组件。

<h2 id="q-012">面试问题：SMPL 的形状参数、姿态参数和输出网格如何理解？</h2>

可以把 SMPL 理解为一个函数：

$$
M(\beta, \theta) \rightarrow (V, J)
$$

其中：

- $\beta$ 控制 body shape。
- $\theta$ 控制 body pose。
- $V$ 是人体 Mesh 顶点。
- $J$ 是关节点位置。

更具体地说：

1. **模板人体**

   SMPL 从一个标准 T-pose 或 rest-pose 模板网格出发。

2. **形状变形**

   根据 $\beta$ 对模板顶点做形状偏移，生成不同体型。

3. **姿态修正**

   根据 $\theta$ 增加 pose-dependent corrective blend shapes，修正关节弯曲导致的塌陷和不自然变形。

4. **骨骼蒙皮**

   使用 LBS 将 rest-pose 顶点变换到目标姿态。

5. **输出网格和关节**

   得到可渲染的人体表面和可用于动作分析的关节点。

<h2 id="q-013">面试问题：Blend Shape 和 LBS 在 SMPL 中分别解决什么问题？</h2>

**Blend Shape** 负责“形变”，**LBS** 负责“摆姿态”。

SMPL 中常见变形可以写成：

$$
T_P(\beta,\theta) = \bar{T} + B_S(\beta) + B_P(\theta)
$$

其中：

- $\bar{T}$ 是模板人体。
- $B_S(\beta)$ 是 shape blend shape，表示体型变化。
- $B_P(\theta)$ 是 pose blend shape，表示姿态导致的局部修正。

然后通过线性混合蒙皮把顶点变换到目标姿态：

$$
v_i' = \sum_k w_{ik}G_k(\theta, J(\beta))v_i
$$

其中：

- $w_{ik}$ 是第 $i$ 个顶点受第 $k$ 个关节影响的权重。
- $G_k$ 是第 $k$ 个关节的全局变换矩阵。
- $J(\beta)$ 是由体型决定的关节位置。

**LBS 的问题：**

传统 LBS 容易出现关节塌陷、糖纸扭曲等伪影。SMPL 通过 pose blend shapes 学习姿态相关修正，使弯肘、抬腿、扭腰等动作更加自然。

<h2 id="q-014">面试问题：SMPL、SMPL-H、SMPL-X、MANO、FLAME 如何对比？</h2>

| 模型 | 建模对象 | 重点能力 | 典型用途 |
| --- | --- | --- | --- |
| SMPL | 身体 | 体型和身体姿态 | 3D 人体重建、动作捕捉、人体生成 |
| MANO | 手 | 手部关节和手型 | 手势估计、手物交互 |
| FLAME | 头部/脸部 | 表情、下颌、眼球、脖子 | 头部 Avatar、Talking Head |
| SMPL-H | 身体 + 手 | 身体动作和手势 | 全身手势动画 |
| SMPL-X | 身体 + 手 + 脸 | 全身、手部和表情统一建模 | 表达性数字人、全身 Avatar |

SMPL-X 可以看作把 SMPL、MANO、FLAME 的能力统一到一个全身模型中。它适合需要同时控制身体动作、手势和面部表情的数字人场景，例如虚拟主播、手语数字人、全身交互角色。

<h2 id="q-015">面试问题：单目 RGB 图像重建 3D 人体为什么困难？</h2>

单目 RGB 人体重建比人脸重建更困难，因为人体具有更强的非刚性变化、遮挡和动作多样性。

主要挑战包括：

1. **深度和尺度歧义**

   单张图像无法唯一确定人体到相机的距离、真实身高和肢体前后关系。

2. **自遮挡和外部遮挡**

   侧身、交叉手臂、坐姿、多人遮挡都会让关键点不可见。

3. **极端姿态**

   舞蹈、运动、瑜伽等动作可能超出训练集常见姿态分布。

4. **宽松衣物**

   图像轮廓通常是衣服轮廓，而 SMPL 表示的是近似裸体或紧身衣人体。

5. **手部和脸部细节**

   全身图像中手和脸分辨率低，容易估计不准。

6. **相机模型误差**

   弱透视相机在近距离、大透视变化场景中可能不够准确。

常见解决思路：

- 引入 2D/3D 关键点监督、轮廓监督、深度监督和人体先验。
- 使用视频时序约束减少抖动。
- 使用多视角或 RGB-D 数据提升可观测性。
- 用 SMPL-X 表示手脸细节。
- 对衣物使用独立 garment layer、隐式场或扩散模型补充外观。

<h2 id="q-016">面试问题：参数化人体模型为什么难以表示宽松衣物？</h2>

SMPL/SMPL-X 的核心假设是：人体表面可以用统一拓扑的低维参数表示。这对身体几何非常有效，但对宽松衣物有天然限制。

主要瓶颈：

1. **拓扑限制**

   宽松衣物可能与身体分离，例如裙摆、外套、袖口、披风；而 SMPL 是单层人体表面。

2. **动态细节缺失**

   布料会产生褶皱、摆动、碰撞和惯性，低维 pose/shape 参数难以完整表示。

3. **物理一致性不足**

   SMPL 的变形主要来自统计学习和骨骼蒙皮，不显式模拟重力、风、碰撞和摩擦。

4. **轮廓与人体不一致**

   图像中的可见轮廓往往是衣服，不是人体。直接拟合 SMPL 容易把宽松衣服误解释为更胖的身体。

改进路线：

- **分层表示**：body mesh + garment mesh。
- **衣物模板**：针对 T-shirt、dress、coat 等建立可变形衣物模板。
- **物理模拟**：用布料仿真或物理约束网络提升动态合理性。
- **隐式表示**：用 SDF/Occupancy/Neural Field 表示拓扑更自由的衣物几何。
- **生成式细节增强**：用 diffusion/GAN/neural texture 生成褶皱和纹理细节。

<h1 id="q-017">4. 3D 数据表示为什么会影响算法设计？</h1>

3D 数据表示决定了模型能否高效学习、是否便于渲染、能否表达拓扑变化、是否适合实时系统。

同一个数字人可以有多种表示：

- 用 **Mesh** 表示可动画人体。
- 用 **点云** 表示扫描结果。
- 用 **SDF** 表示连续曲面。
- 用 **NeRF** 表示视角相关辐射场。
- 用 **3D Gaussian Splatting** 表示可快速渲染的显式辐射场。
- 用 **多视角图像** 表示训练监督。
- 用 **RGB-D** 表示带深度的相机观测。

没有一种表示适合所有任务。面试中要从表达能力、可微性、渲染效率、内存成本、拓扑灵活性、动画控制和工程部署几个角度回答。

<h2 id="q-018">面试问题：点云、Mesh、体素、RGB-D、多视角图像如何对比？</h2>

| 表示 | 优点 | 缺点 | 数字人用途 |
| --- | --- | --- | --- |
| 点云 | 简单、直接来自扫描、无需拓扑 | 缺面片连接，渲染和动画不方便 | 三维扫描、粗重建、3DGS 初始化 |
| Mesh | 可渲染、可绑定骨骼、可编辑、引擎友好 | 拓扑固定，难表示头发/衣物复杂结构 | Avatar 资产、游戏角色、虚拟主播 |
| 体素 | 规则网格，适合 3D CNN | 分辨率高时内存爆炸 | 粗几何、医学、占用预测 |
| RGB-D | 有颜色和深度，采集方便 | 深度噪声、遮挡不可见 | 动作捕捉、实时重建、AR |
| 多视角图像 | 保留真实外观，适合神经渲染 | 需要标定，相机覆盖影响质量 | NeRF/3DGS 训练、多视角 Avatar |

简洁回答：

- 要实时动画：优先 Mesh + 骨骼。
- 要高保真新视角：优先 NeRF/3DGS。
- 要连续几何重建：优先 SDF/Occupancy。
- 要工程采集：RGB-D 和多视角图像常作为输入。

<h2 id="q-019">面试问题：SDF 和 Occupancy Field 有什么区别？</h2>

SDF 和 Occupancy Field 都是隐式三维表示，用函数而不是显式面片描述物体表面。

**SDF，Signed Distance Function：**

$$
f(x) = d
$$

其中 $d$ 表示点 $x$ 到最近表面的有符号距离。通常约定：

- $f(x) < 0$：点在物体内部。
- $f(x) = 0$：点在物体表面。
- $f(x) > 0$：点在物体外部。

表面可以表示为：

$$
\mathcal{S} = \{x \mid f(x)=0\}
$$

**Occupancy Field：**

$$
o(x) \in [0,1]
$$

它表示点 $x$ 被物体占据的概率。决策边界通常对应物体表面。

对比：

| 维度 | SDF | Occupancy |
| --- | --- | --- |
| 输出 | 有符号距离 | 占用概率 |
| 几何信息 | 包含距离和内外方向 | 主要表示内外分类 |
| 表面提取 | zero level set | 阈值面 |
| 优势 | 法线、距离约束更自然 | 分类训练直观 |
| 难点 | 距离监督和 Eikonal 约束要求高 | 几何精度和表面细节可能不足 |

在数字人中，SDF 常用于高质量人体/衣物几何，Occupancy 常用于人体占用预测和从图像恢复可变拓扑形状。

<h2 id="q-020">面试问题：NeRF 的核心思想是什么？</h2>

NeRF，全称 Neural Radiance Fields，是一种用神经网络表示三维场景辐射场的方法。它学习一个函数：

$$
F_\theta(x, d) = (\sigma, c)
$$

其中：

- $x$ 是三维空间位置。
- $d$ 是观察方向。
- $\sigma$ 是体密度。
- $c$ 是该位置在该方向下的颜色。

对一条相机射线 $r(t)$，像素颜色通过体渲染积分得到：

$$
C(r)=\int_{t_n}^{t_f} T(t)\sigma(r(t))c(r(t),d)dt
$$

$$
T(t)=\exp\left(-\int_{t_n}^{t}\sigma(r(s))ds\right)
$$

NeRF 的核心价值是：用多视角图像监督一个连续三维函数，从而实现高质量新视角合成。

数字人场景中的 NeRF 常见变体：

- **静态头部 NeRF**：适合高保真人头新视角。
- **动态 NeRF**：增加时间或形变场，表示表情和动作。
- **条件 NeRF**：用音频、表情参数、骨骼或 SMPL 条件控制。
- **Avatar NeRF**：将人物绑定到 canonical space，再通过形变场映射到观察空间。

<h2 id="q-021">面试问题：3D Gaussian Splatting 为什么适合实时数字人？</h2>

3D Gaussian Splatting，常简称 3DGS，是一种显式辐射场表示。它用大量三维高斯椭球表示场景，每个 Gaussian 通常包含：

- 三维位置。
- 尺度和旋转，或协方差矩阵。
- 不透明度。
- 颜色或球谐系数。

渲染时将三维 Gaussian 投影到屏幕空间，通过 splatting 和 alpha blending 快速合成图像。

相比 NeRF，3DGS 的优势是：

- **渲染速度快**：更适合实时预览和交互。
- **显式表示**：不需要对每条射线密集查询 MLP。
- **训练和编辑更直接**：可以增删、移动、剪枝 Gaussian。
- **高保真新视角**：对真实捕获场景能保留丰富纹理。

数字人中的典型用法：

- 多视角采集真人头像或上半身，训练可实时渲染的 3DGS Avatar。
- 将 3DGS 与骨骼、SMPL 或 FLAME 绑定，实现动态驱动。
- 用 3DGS 做高保真头发、衣物和非刚体外观补充。

局限也要说清楚：

- 动态人体需要处理 Gaussian 的形变和拓扑变化。
- 直接编辑语义、骨骼绑定和碰撞仍不如 Mesh 成熟。
- 训练质量依赖相机标定、多视角覆盖和数据稳定性。
- 透明区域、反光材质、快速运动和遮挡会增加难度。

<h2 id="q-022">面试问题：Mesh Avatar、NeRF Avatar、3DGS Avatar 各自适合什么场景？</h2>

| 类型 | 优势 | 劣势 | 适合场景 |
| --- | --- | --- | --- |
| Mesh Avatar | 可编辑、可绑定骨骼、引擎友好、实时稳定 | 真实感依赖建模/贴图/材质，头发衣物复杂 | 游戏、AR/VR、虚拟主播、工业实时交互 |
| NeRF Avatar | 真实感强，视角相关效果好，适合真实捕获 | 渲染慢，动态控制复杂，编辑性弱 | 高保真重放、离线渲染、研究型 Avatar |
| 3DGS Avatar | 渲染快，真实感好，适合实时新视角 | 动态绑定、语义编辑和物理交互仍复杂 | 实时高保真头像、空间视频、交互式展示 |

工程选型建议：

- 需要稳定上线、低延迟、多平台：优先 Mesh/SMPL/BlendShape。
- 需要真实捕获和高保真外观：考虑 NeRF/3DGS。
- 需要兼顾可控和真实：使用参数化模型控制运动，用神经渲染或扩散模型增强外观。

<h1 id="q-023">5. 传统渲染管线包括哪些核心步骤？</h1>

渲染是从三维场景生成二维图像的过程。传统图形学中，一个简化的渲染管线可以表示为：

$$
I = R(M, A, P, L)
$$

其中：

- $M$：几何模型，如 Mesh、顶点、三角面、法线。
- $A$：外观属性，如纹理、材质、反照率、粗糙度。
- $P$：相机参数，如内参、外参、投影矩阵。
- $L$：光照条件，如点光、环境光、HDRI、球谐光照。
- $R$：渲染器。
- $I$：输出图像。

典型步骤：

1. **模型变换**

   将物体从局部坐标系变换到世界坐标系。

2. **视图变换**

   将世界坐标变换到相机坐标系。

3. **投影变换**

   将三维点投影到二维屏幕空间。

4. **裁剪与光栅化**

   判断三角形覆盖哪些像素。

5. **深度测试**

   用 Z-buffer 保留离相机最近的表面。

6. **着色**

   根据法线、材质、纹理和光照计算像素颜色。

7. **后处理**

   包括抗锯齿、色调映射、阴影、景深、运动模糊等。

<h2 id="q-024">面试问题：局部光照和全局光照有什么区别？</h2>

**局部光照模型** 只考虑光源直接照射到物体表面的效果，不考虑物体之间多次反射。常见模型包括 Lambert、Phong、Blinn-Phong 等。

Lambert 漫反射可以写成：

$$
I_d = k_d I_l \max(0, n \cdot l)
$$

其中：

- $n$ 是表面法线。
- $l$ 是光照方向。
- $k_d$ 是漫反射系数。
- $I_l$ 是光源强度。

**全局光照模型** 会考虑光在场景中的多次传播，例如阴影、间接光、反射、折射和颜色溢出。常见方法包括路径追踪、光线追踪、辐射度法、光子映射等。

对比：

| 维度 | 局部光照 | 全局光照 |
| --- | --- | --- |
| 计算成本 | 低 | 高 |
| 真实感 | 中等 | 高 |
| 是否考虑间接光 | 通常不考虑 | 考虑 |
| 实时性 | 容易实时 | 需要加速或近似 |
| 数字人用途 | 实时 Avatar、移动端 | 影视级渲染、高质量离线资产 |

数字人系统中，实时交互通常采用近似 PBR 和预计算光照；影视级或高质量数据生成可以用路径追踪。

<h2 id="q-025">面试问题：光栅化和 Z-buffer 为什么会带来不可微问题？</h2>

可微渲染的核心难点来自传统渲染中的离散决策。

**光栅化** 会判断一个像素是否被某个三角形覆盖。这个判断近似是二值函数：

$$
m(p, t) \in \{0,1\}
$$

当像素 $p$ 在三角形 $t$ 内部时为 1，否则为 0。顶点轻微移动时，大多数像素的覆盖关系不变；但跨过边界时覆盖关系突然变化。因此该过程对顶点位置不是处处连续可导。

**Z-buffer** 会在多个三角形覆盖同一像素时选择深度最近的那个：

$$
t^* = \arg\min_t z_t
$$

这个 argmin 也是离散选择。当两个三角形深度接近并发生前后切换时，像素颜色会突变。

所以，传统渲染中虽然模型变换、投影、光照计算大多可微，但光栅化的覆盖判断和 Z-buffer 的可见性选择会破坏端到端求导。

<h2 id="q-026">面试问题：可微渲染是什么？</h2>

可微渲染是指渲染结果可以对输入场景参数求导的渲染过程。它既要完成正向渲染：

$$
I = R(\theta)
$$

也要支持反向传播：

$$
\frac{\partial I}{\partial \theta}
$$

其中 $\theta$ 可以是：

- 顶点位置。
- 相机参数。
- 姿态参数。
- 形状参数。
- 表情参数。
- 材质和纹理。
- 光照参数。
- 隐式场参数。

可微渲染的意义是：可以通过二维图像监督来优化三维参数。例如给定真实图像 $I_{\text{gt}}$，渲染预测图像 $I_{\text{render}}$，最小化：

$$
\mathcal{L} = \|I_{\text{render}} - I_{\text{gt}}\|_1
$$

就可以反向更新 3D 模型、相机、姿态和光照。

<h2 id="q-027">面试问题：可微渲染有哪些主流实现思路？</h2>

主流思路可以分为三类：

1. **近似梯度**

   正向渲染保持传统光栅化流程，但在反向传播时构造近似梯度。优点是正向结果真实、兼容传统管线；缺点是梯度设计复杂，可能依赖任务。

2. **软光栅化**

   把三角形覆盖从硬 0/1 判断变成连续概率。例如像素离三角形越近，贡献越大；深度选择也从 hard argmin 改成 soft aggregation。优点是端到端可导；缺点是渲染结果可能变软或出现透明边界。

3. **神经渲染/体渲染**

   使用 NeRF、SDF、3DGS 或神经纹理等连续表示，通过可微采样和合成得到图像。优点是适合高保真外观和隐式几何；缺点是训练、加速和编辑复杂。

常见工具和代表方向包括：

- Neural Mesh Renderer。
- Soft Rasterizer。
- PyTorch3D differentiable renderer。
- nvdiffrast。
- NeRF/NeuS/VolSDF 类体渲染。
- 3DGS 的可微 splatting。

面试回答时要注意：可微渲染不是一种固定算法，而是一类让图像损失能反向约束三维场景参数的方法。

<h2 id="q-028">面试问题：可微渲染如何用于 3DMM/SMPL 参数优化？</h2>

以人脸 3DMM 为例，目标是从输入图像 $I$ 中估计参数：

$$
\theta = \{\alpha, \beta, \gamma, R, t, L\}
$$

其中包括身份、表情、纹理、姿态、平移和光照。流程如下：

1. 用当前参数生成 3D 人脸 Mesh。
2. 通过可微渲染器得到渲染图、mask、depth 或 landmark。
3. 计算损失：

$$
\mathcal{L} =
\lambda_{\text{photo}}\mathcal{L}_{\text{photo}} +
\lambda_{\text{lm}}\mathcal{L}_{\text{landmark}} +
\lambda_{\text{id}}\mathcal{L}_{\text{id}} +
\lambda_{\text{reg}}\mathcal{L}_{\text{reg}}
$$

4. 反向传播更新参数。

SMPL 优化也类似，常见损失包括：

- 2D keypoint reprojection loss。
- silhouette/mask loss。
- depth/normal loss。
- pose prior。
- shape prior。
- joint angle prior。
- temporal smoothness。
- collision loss。

可微渲染的优势是可以利用大量 2D 图像监督三维参数；风险是图像损失容易被光照、遮挡、纹理和局部最优干扰，所以需要先验和鲁棒损失。

<h1 id="q-029">6. 3D 数字人与生成式模型如何结合？</h1>

AIGC 时代的 3D 数字人越来越多采用“3D 结构控制 + 生成式外观增强”的混合路线。

常见结合方式：

1. **3D 条件控制扩散模型**

   先从 3DMM/SMPL/SMPL-X 渲染 depth、normal、pose、segmentation、DensePose、UV，再作为 ControlNet/Adapter/视频扩散模型条件。

2. **生成模型补全 3D 资产**

   使用图像/文本到 3D、纹理生成、材质生成、法线生成，为 Mesh Avatar 生成外观。

3. **神经渲染增强真实感**

   用 NeRF/3DGS/neural texture 表示传统 Mesh 难以表达的头发、皮肤细节、衣物褶皱和视角相关反射。

4. **3D 约束视频生成**

   通过相机轨迹、骨架动作、表情曲线和口型参数约束视频生成，减少时序漂移。

5. **从视频反向构建 Avatar**

   从一段真人视频估计人脸/人体参数、纹理、动态细节，再训练可驱动 Avatar。

一句话总结：

3D 提供结构和控制，生成模型提供真实感和多样性；二者结合是当前数字人落地的主流方向。

<h2 id="q-030">面试问题：如何设计一个可控 3D 数字人生成系统？</h2>

可以按“输入、建模、驱动、渲染、增强、评估、部署”回答。

**1. 输入层**

- 身份参考：单图、多图或短视频。
- 驱动信号：文本、音频、动作视频、骨骼序列、表情参数。
- 场景条件：相机、背景、光照、服装、风格。

**2. 建模层**

- 人脸：检测关键点，拟合 FLAME/3DMM，提取身份纹理和表情空间。
- 人体：估计 SMPL/SMPL-X 参数，得到骨架、体型和姿态。
- 外观：构建 Mesh 贴图、神经纹理、NeRF 或 3DGS 表示。

**3. 驱动层**

- 音频到口型和表情。
- 文本到动作或手势。
- 视频到姿态迁移。
- 手动控制表情、视线、头部和身体动作。

**4. 渲染层**

- 实时需求：Mesh + PBR + BlendShape/LBS。
- 高保真需求：NeRF/3DGS/neural rendering。
- 生成增强：渲染 depth/normal/segmentation，再用扩散模型提升真实感。

**5. 时序层**

- 使用运动平滑、光流一致性、表情曲线平滑、身份一致性损失。
- 对长视频做分段生成和跨段状态继承。

**6. 部署层**

- 模型压缩、缓存、流式推理、GPU/CPU 分工。
- Web/移动端使用轻量 Mesh，云端使用高保真生成。
- 增加安全审核、水印和授权校验。

<h2 id="q-031">面试问题：3D 数字人如何评估质量？</h2>

3D 数字人的评估要同时看几何、外观、运动、身份和工程性能。

| 评估维度 | 关注点 | 常见指标或方法 |
| --- | --- | --- |
| 几何精度 | 脸型、体型、深度、法线是否准确 | Chamfer Distance、P2S、normal consistency、关键点误差 |
| 身份保持 | 是否像目标人物 | 人脸识别相似度、人工偏好评测 |
| 表情/口型 | 是否和音频、情绪一致 | LSE-C/LSE-D、landmark error、用户评分 |
| 姿态动作 | 骨架是否合理、是否抖动 | MPJPE、PA-MPJPE、加速度误差、时序平滑 |
| 纹理真实感 | 皮肤、头发、衣物是否真实 | FID、LPIPS、CLIP-IQA、人工评测 |
| 视角一致性 | 换视角是否稳定 | 多视角重投影误差、novel view PSNR/SSIM/LPIPS |
| 渲染性能 | 是否满足实时 | FPS、延迟、显存、模型大小 |
| 可编辑性 | 能否改姿态、表情、服装、光照 | 编辑成功率、属性保持率 |

面试中可以补一句：真实产品不能只看单帧指标。数字人更需要看长视频稳定性、交互延迟、异常姿态鲁棒性和用户主观感受。

<h2 id="q-032">面试问题：线上 3D 数字人系统常见问题如何排查？</h2>

常见问题和排查方向：

| 问题 | 可能原因 | 排查/优化 |
| --- | --- | --- |
| 脸不像本人 | 身份参数不准、纹理受光照污染、参考图质量差 | 多图融合、ID loss、光照分离、正脸高质量参考 |
| 嘴型不准 | 音频特征弱、表情映射延迟、下颌/唇部模型不足 | 音素级对齐、口型曲线平滑、增加唇部 loss |
| 头部抖动 | landmark 检测抖动、逐帧拟合无时序约束 | Kalman/OneEuro filter、时序优化、多帧网络 |
| 手部畸形 | 分辨率低、遮挡多、手部先验弱 | 使用 SMPL-X/MANO、手部 crop、手势先验 |
| 衣物穿模 | body-cloth 未分离、无碰撞约束 | garment layer、碰撞 loss、物理模拟 |
| 视角切换不一致 | 2D 生成缺三维约束、纹理空间不稳定 | 使用 Mesh/NeRF/3DGS 或 UV 一致性 |
| 渲染延迟高 | 模型过大、神经渲染采样多、后处理复杂 | LOD、剪枝、缓存、TensorRT、分辨率自适应 |
| 皮肤塑料感 | 材质/光照不真实，缺少次表面散射 | PBR 校准、环境光照、皮肤 shader、生成式增强 |
| 长视频漂移 | 分段状态不一致、身份条件衰减 | 跨段记忆、参考帧约束、时序一致性 loss |

工程排查顺序建议：

1. 先确认输入质量：检测框、关键点、音频、相机标定是否稳定。
2. 再确认参数估计：3DMM/SMPL/FLAME 参数是否合理。
3. 再看渲染条件：相机、光照、纹理、mask、深度是否匹配。
4. 最后看生成增强：扩散模型是否引入身份漂移、闪烁或局部幻觉。

## 高频速记

1. 3D 数字人的核心价值是可控、可解释、可实时、跨视角一致。
2. 3DMM 用低维统计空间表示人脸形状、表情和纹理。
3. BFM 偏经典人脸统计模型，FLAME 偏完整头部和表情动画。
4. SMPL 用 shape、pose、blend shape 和 LBS 表示可动画人体。
5. SMPL-X 统一身体、手部和脸部，适合表达性全身数字人。
6. 宽松衣物、头发、遮挡和高频皮肤细节是参数化模型的主要短板。
7. Mesh 适合实时动画，SDF/Occupancy 适合连续几何，NeRF/3DGS 适合高保真新视角。
8. NeRF 是隐式体渲染，3DGS 是显式高斯 splatting，后者更适合实时。
9. 光栅化和 Z-buffer 的离散选择是传统渲染不可微的关键原因。
10. 可微渲染让 2D 图像损失可以反向优化 3D 几何、相机、姿态、材质和光照。
11. AIGC 数字人主流路线是 3D 结构控制加生成式外观增强。
