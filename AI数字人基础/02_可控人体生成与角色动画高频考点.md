<h1 id="q-001">1. 可控人体生成的核心目标是什么？</h1>

可控人体生成的目标是：给定一个人物外观参考和一个控制条件，让模型生成该人物在目标动作、姿态、镜头或场景下的图像/视频，同时保持身份、服装、发型、配饰、背景和时序稳定。

典型输入输出可以写成：

$$
V_{\text{out}} = G(I_{\text{ref}}, C_{\text{motion}}, C_{\text{text}}, C_{\text{scene}})
$$

其中：

- $I_{\text{ref}}$：参考人物图像或参考视频。
- $C_{\text{motion}}$：姿态、骨架、DensePose、SMPL、驱动视频。
- $C_{\text{text}}$：文本描述，如动作、场景、风格。
- $C_{\text{scene}}$：背景、镜头、光照、环境约束。

**面试回答重点：**

可控人体生成不是单纯“生成人”，而是要在“强控制”和“高真实感”之间平衡。控制太弱，动作不跟随；控制太强，画面僵硬、身份丢失或服装纹理变形。

<h2 id="q-002">面试问题：人体驱动常见输入条件有哪些？</h2>

人体驱动常见条件包括：

- **2D Pose / OpenPose / DWPose**：人体骨架、手部、脸部关键点。
- **DensePose**：人体表面 UV 对应关系，能提供比骨架更密集的身体区域信息。
- **Depth**：深度图，提供前后空间关系。
- **Normal**：法线图，提供局部表面朝向。
- **Segmentation / Parsing**：人体、衣服、头发、手臂等语义区域。
- **SMPL/SMPL-X**：参数化人体模型，提供 3D 姿态、体型、手势、表情。
- **Driving Video**：直接从目标视频提取动作、头动、表情和镜头运动。

**选择原则：**

- 只需要粗动作：2D Pose 足够。
- 需要身体表面贴合：DensePose 更强。
- 需要 3D 视角/体型一致：SMPL/SMPL-X 更合适。
- 需要服装/遮挡处理：Parsing、Depth、Normal 往往要一起用。

<h2 id="q-003">面试问题：Pose、DensePose、Depth、Normal、SMPL/SMPL-X 各自适合什么场景？</h2>

| 条件 | 优点 | 缺点 | 适合场景 |
| --- | --- | --- | --- |
| 2D Pose | 简单、稳定、易获取 | 缺少体表和深度信息 | 舞蹈、全身动作驱动 |
| DensePose | 身体表面对应更密集 | 对遮挡和衣物敏感 | 服装贴合、人体表面驱动 |
| Depth | 提供前后关系 | 单目深度可能不准 | 遮挡、空间布局控制 |
| Normal | 表面朝向清晰 | 对估计质量依赖高 | 3D 感、光照和结构约束 |
| SMPL | 3D 姿态和体型可控 | 难表达宽松衣物和细节 | 3D 动作、骨骼驱动 |
| SMPL-X | 身体+手+脸统一 | 拟合和数据要求更高 | 数字人、手势、表情动画 |

**面试易错点：**

控制条件越多不一定越好。多条件之间可能信息冗余或冲突，例如 DensePose 和原图衣服轮廓可能泄露目标服装形状，导致模型在换衣时填满错误区域。

<h2 id="q-004">面试问题：基于图像扩散模型做人像视频动画的典型结构是什么？</h2>

典型结构是“图像扩散底模 + 参考外观分支 + 运动控制分支 + 时序模块”。

可抽象为：

$$
\epsilon_\theta(z_t,t,c) = \epsilon_\theta(z_t,t,c_{\text{text}},c_{\text{ref}},c_{\text{pose}},c_{\text{motion}})
$$

其中：

- $c_{\text{ref}}$：参考人物图像特征。
- $c_{\text{pose}}$：目标姿态序列。
- $c_{\text{motion}}$：时序运动先验。

训练目标通常仍是扩散噪声预测：

$$
L=\mathbb{E}\left[\|\epsilon-\epsilon_\theta(z_t,t,c)\|_2^2\right]
$$

**常见模块组合：**

- SD1.5/SDXL U-Net
- ReferenceNet
- Pose Guider / ControlNet
- AnimateDiff Motion Module
- CLIP image encoder / face encoder
- temporal attention / 3D convolution

<h1 id="q-005">2. Animate Anyone 类方法为什么成为可控人体动画的经典范式？</h1>

Animate Anyone 类方法解决了一个关键问题：如何让参考人物在动作变化中保持外观一致。它把“外观”和“动作”解耦，用参考图提供身份和纹理，用姿态序列提供动作，用时序模块保证视频连贯。

<h2 id="q-006">面试问题：ReferenceNet 如何保持人物外观一致？</h2>

ReferenceNet 通常使用与扩散 U-Net 类似的结构处理参考图像，输出多尺度特征：

$$
F_{\text{ref}}^{(l)} = R^{(l)}(I_{\text{ref}})
$$

在去噪 U-Net 的对应层中注入：

$$
F_{\text{denoise}}^{(l)} \leftarrow \text{Fuse}(F_{\text{denoise}}^{(l)},F_{\text{ref}}^{(l)})
$$

常见融合方式：

- attention 读参考特征
- feature concat
- residual addition
- AdaIN/调制
- reference attention

**为什么比 CLIP 全局 embedding 更强？**

CLIP embedding 更偏语义，能知道“这是一个穿红裙的人”，但不擅长保存衣服纹理、局部配饰、发型边缘等空间细节。ReferenceNet 保留多尺度空间特征，因此更适合高保真角色动画。

<h2 id="q-007">面试问题：Pose Guider / Pose Encoder 如何影响动作跟随能力？</h2>

Pose Encoder 把姿态图编码到 latent 空间：

$$
F_{\text{pose}} = E_{\text{pose}}(P_{1:T})
$$

再注入到视频生成主干。

影响动作跟随的因素：

- 姿态检测质量：关键点错、漏、抖会直接影响生成。
- 姿态表示密度：骨架稀疏，DensePose 更密集。
- 时序平滑：姿态序列抖动会带来视频抖动。
- 注入层级：浅层影响结构，深层影响细节。
- 训练数据覆盖：极端动作和大幅转身需要足够数据。

**工程建议：**

姿态条件通常需要先做时序滤波、置信度过滤、缺失点补全，尤其是手部和脸部关键点。

<h2 id="q-008">面试问题：Motion Module 如何提升帧间一致性？</h2>

如果逐帧生成，每一帧的噪声采样和去噪路径不同，容易出现闪烁。Motion Module 在时间维度上建模多帧关系：

$$
Z \in \mathbb{R}^{B \times C \times T \times H \times W}
$$

它可以是：

- temporal attention
- 3D convolution
- temporal transformer
- motion LoRA / adapter

其作用：

1. 学习自然运动先验。
2. 让相邻帧共享时序上下文。
3. 减少局部纹理跳变。
4. 让姿态变化更平滑。

**面试要点：**

Motion Module 不是负责“人物长什么样”，而是负责“连续帧怎么动、怎么稳定”。外观保持主要靠 ReferenceNet/ID 条件，动作跟随主要靠 Pose Encoder，时序稳定靠 Motion Module。

<h2 id="q-009">面试问题：为什么人体驱动中的脸部 ID 最容易崩？</h2>

原因包括：

- 脸部在全身图中占比小，训练损失容易被身体和背景主导。
- 人脸对微小差异敏感，眼睛、鼻子、嘴角轻微变化都会被察觉。
- 姿态和表情变化会改变局部几何，模型容易把身份和表情混淆。
- ReferenceNet 保存外观，但不一定有强身份判别能力。
- 视频生成中累积误差会让脸逐渐漂移。

常见解决方式：

- 使用 ArcFace/InsightFace ID loss。
- 加 Face Adapter 或局部人脸 ReferenceNet。
- 对脸部区域加权训练。
- 生成后做人脸局部修复。
- 多参考图提取稳定 ID embedding。
- 使用 face parsing mask 限制身份条件作用区域。

<h1 id="q-010">3. OmniHuman、StableAnimator、SkyReels-A1、Wan-Animate 代表了什么趋势？</h1>

这些新方法代表从“单任务、单条件、U-Net 视频扩散”走向“视频基础模型 + 多条件控制 + DiT/Flow + 统一角色动画”的趋势。它们更重视大规模数据、多条件混合训练、身份一致性、全身动作、长视频和工程可用性。

<h2 id="q-011">面试问题：OmniHuman 的多条件混合训练为什么有效？</h2>

OmniHuman 的关键思想是：不要只用严格过滤后的音频-视频数据，而是把文本、图像、音频、姿态等多种条件和多种数据都纳入训练。

可抽象为多任务条件生成：

$$
V = G(I_{\text{ref}}, c_{\text{text}}, c_{\text{audio}}, c_{\text{pose}})
$$

不同训练样本可能只激活部分条件：

- 图像 + 文本
- 图像 + 音频
- 图像 + 姿态
- 图像 + 文本 + 音频 + 姿态

**为什么有效？**

- 弱条件数据更多，可扩大训练规模。
- 强条件数据更精确，但数量少。
- 混合训练让模型既保留基础视频生成能力，又学会音频/姿态控制。
- 对全身动画和手势生成更友好。

**面试重点：**

多条件训练的难点在于条件比例和条件强度。强条件比例过高，模型可能丧失开放生成能力；弱条件比例过高，控制能力不足。

<h2 id="q-012">面试问题：StableAnimator 如何解决身份一致性？</h2>

StableAnimator 关注高质量、身份一致的人像动画。其核心思路包括：

1. **全局内容感知面部编码器**：不仅提取脸部 ID，还感知参考图整体布局，避免脸部特征和整体画面割裂。
2. **分布感知 ID Adapter**：让 ID embedding 与扩散 latent 的特征分布对齐，降低直接注入导致的空间失真。
3. **推理时面部优化**：通过额外优化过程增强脸部身份和质量。

**面试表达：**

StableAnimator 的重点不是单纯把 ArcFace embedding 塞进模型，而是解决“ID 特征如何与视频扩散模型的 latent 分布对齐”的问题。

<h2 id="q-013">面试问题：SkyReels-A1 如何用表达感知关键点做肖像动画？</h2>

SkyReels-A1 面向高质量肖像动画，强调 expression-aware landmarks。它不是只用普通 2D landmark，而是希望捕捉更精细的 3D 面部表情变化。

典型流程：

1. 从驱动视频提取表情感知关键点。
2. 通过 3D/因果编码器把关键点编码到视频 latent 对齐空间。
3. 使用 DiT 视频生成主干生成目标肖像动画。
4. 使用面部图像-文本对齐模块增强身份一致性。

**优势：**

- 比纯音频驱动更可控。
- 比普通 landmark 更能表达细微表情。
- DiT 视频底模有更强生成质量和时序建模能力。

<h2 id="q-014">面试问题：Wan-Animate 的整体复制和角色替换如何理解？</h2>

Wan-Animate 的核心思想可以概括为“整体复制”：尽可能完整保留源角色的身份、服装、发型、配饰和细节，同时把这些外观放到驱动视频的动作和环境中。

它统一两个任务：

- **角色动画**：源角色 + 驱动姿态 -> 源角色动起来。
- **角色替换**：源角色 + 目标视频 -> 替换目标视频中的角色。

关键模块：

- Body Adapter：编码人体姿态和动作。
- Face Adapter：编码面部表情和细节。
- Relighting LoRA：让替换角色适应目标环境光照。
- Pose Retargeting：处理源角色和驱动角色体型差异。

**面试要点：**

角色替换比角色动画更难，因为它不仅要跟动作，还要和原视频环境、遮挡、光照、镜头运动保持一致。

<h1 id="q-015">4. 可控人体动画如何设计训练数据和损失？</h1>

训练数据和损失设计决定模型是否真正学会“外观-动作解耦”。如果只做同帧或同视频重建，模型容易 copy-paste；如果跨身份、跨姿态数据太难，模型又容易身份漂移。

<h2 id="q-016">面试问题：自重构和交叉重构分别训练什么能力？</h2>

**自重构（Self-Reconstruction）**：

从同一个视频中取参考帧和目标帧：

$$
(I_{\text{ref}}^A, P_{\text{target}}^A) \rightarrow I_{\text{target}}^A
$$

它训练模型保持同一身份和同一场景下的动作迁移能力。

**交叉重构（Cross-Reconstruction）**：

从不同视频/不同身份中组合参考外观和目标姿态：

$$
(I_{\text{ref}}^A, P_{\text{target}}^B) \rightarrow \tilde{I}
$$

它训练模型把任意外观和任意姿态组合，提升泛化能力。

**两者配合：**

自重构稳定训练，交叉重构避免死记硬背。比例上通常需要从自重构为主逐渐加入交叉任务。

<h2 id="q-017">面试问题：如何避免条件冗余和 Copy-Paste？</h2>

条件冗余指多个条件携带重复甚至冲突的信息。例如参考图、DensePose、mask、CLIP image embedding 都可能包含人物外观，导致模型不知道该信谁。

缓解策略：

- 条件解耦：身份、姿态、表情、服装分别编码。
- 条件 dropout：训练中随机丢弃部分条件，避免模型依赖单一路径。
- 延迟注入：前期弱化 ID，后期增强细节。
- 区域限制：ID 条件只影响脸部，服装条件只影响衣服区域。
- 多参考/跨姿态训练：减少单图 copy-paste。
- Copy-Paste 指标：比较生成图更接近参考图还是目标真实图，惩罚无脑复制。

<h2 id="q-018">面试问题：如何处理体型差异、手部动作和遮挡？</h2>

**体型差异：**

- 使用 pose retargeting 调整骨骼比例。
- 参考 T-pose 或标准姿态估计真实骨骼长度。
- 替换模式下谨慎 retarget，避免破坏人与环境交互。

**手部动作：**

- 使用 DWPose/VitPose 提取手部关键点。
- 对手部区域增加损失权重。
- 使用手部关键点置信度和方差评估质量。
- 必要时引入手部局部修复模型。

**遮挡：**

- 使用人体 parsing / SAM 分割层级。
- 建模人物、衣服、手、头发、配饰之间的前后关系。
- 对遮挡区域做局部 inpainting。
- 视频中使用 tracking 保持遮挡关系稳定。
