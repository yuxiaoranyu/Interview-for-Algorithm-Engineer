<h1 id="q-001">1. ID 保持生成的核心目标是什么？</h1>

ID 保持生成的目标是：给定一个或多个参考人物图像，在新文本、新姿态、新场景或新风格条件下生成同一身份的人物图像/视频。

可写成：

$$
I_{\text{out}} = G(c_{\text{text}}, c_{\text{id}}, c_{\text{layout}}, c_{\text{style}})
$$

其中 $c_{\text{id}}$ 是身份条件，通常来自人脸识别模型、CLIP 图像编码器、ReferenceNet、多图聚合 embedding 或专门的 ID adapter。

**关键矛盾：**

- ID 条件太弱：不像本人。
- ID 条件太强：copy-paste、表情僵硬、文本不听话。
- 文本条件太强：身份漂移。
- 结构条件太强：姿态准确但细节丢失。

<h2 id="q-002">面试问题：ID 保持生成、个性化生成、换脸有什么区别？</h2>

**ID 保持生成**：以参考身份为条件生成新图或新视频，可以改变场景、动作、风格、服装。

**个性化生成**：范围更大，不只人脸，也可以是特定角色、物体、IP、宠物、商品。DreamBooth、LoRA、Textual Inversion、PhotoMaker 都属于个性化生成范畴。

**换脸**：在已有目标图像/视频中替换脸部身份，通常要求保留目标姿态、表情、背景、光照和非脸区域。

**区别总结：**

- ID 保持生成：从条件生成整体画面。
- 个性化生成：学习特定主体概念。
- 换脸：局部身份替换和融合。

<h2 id="q-003">面试问题：IP-Adapter、InstantID、PuLID、PhotoMaker 的技术路线如何对比？</h2>

| 方法 | 核心条件 | 注入方式 | 优势 | 风险 |
| --- | --- | --- | --- | --- |
| IP-Adapter | CLIP image feature | 解耦 cross-attention | 通用图像提示，兼容性强 | 人脸 ID 精度有限 |
| InstantID | face ID embedding + landmark | IdentityNet / adapter | 单图强 ID 保持 | 可能影响可编辑性 |
| PuLID | face ID + 对比对齐 | 轻量分支 + ID loss | 尽量保持原 T2I 能力 | 训练设计较复杂 |
| PhotoMaker | 多图 stacked ID embedding | ID embedding 融合 | 多图身份更稳 | 依赖多参考图质量 |

**面试要点：**

IP-Adapter 更像通用 image prompt；InstantID/PuLID 更偏人脸身份；PhotoMaker 通过多图聚合降低单图姿态、光照和表情干扰。

<h2 id="q-004">面试问题：UniPortrait、ConsistentID 如何处理多人或细粒度 ID？</h2>

**UniPortrait** 关注单人和多人 ID 生成。核心包括：

- ID Embedding Module：提取身份相关特征。
- ID Routing Module：为图像不同空间位置路由不同 ID。
- Gumbel-softmax / routing regularization：确保每个 ID 被路由到正确面部区域。

它解决多人生成时“哪个脸对应哪个 ID”的问题。

**ConsistentID** 强调多模态面部提示和细粒度 ID：

- 结合面部特征、面部描述、整体上下文。
- 使用面部分割 mask 约束 face attention。
- 延迟主题条件，在身份保持和可编辑性之间平衡。

**总结：**

UniPortrait 更强调多 ID 路由，ConsistentID 更强调细粒度面部条件和区域注意力对齐。

<h2 id="q-005">面试问题：InfiniteYou、FlashFace、InstantCharacter 代表了什么新趋势？</h2>

这些方法代表 ID/角色定制从 U-Net/SD1.5 走向 DiT、FLUX、大视觉编码器和大规模数据训练。

**InfiniteYou**：面向 DiT/FLUX 的身份保持生成，引入 InfuseNet，通过残差连接把身份特征注入基础 DiT，减少与文本特征冲突。

**FlashFace**：使用 Face ReferenceNet 提取特征图而不是少量 ID token，保留更细的脸型、纹理、疤痕等局部细节，并采用解耦参考注意力。

**InstantCharacter**：面向通用角色图像生成，结合 SigLIP、DINOv2、区域级特征、中间 transformer encoder 和 Q-former 式投影，强调可扩展角色适配器和三阶段训练。

**趋势总结：**

- 从单一全局 ID token 转向多尺度空间特征。
- 从人脸身份转向通用角色一致。
- 从 U-Net adapter 转向 DiT/FLUX 适配。
- 从小数据微调转向千万级角色数据训练。

<h1 id="q-006">2. 为什么高保真身份注入很难？</h1>

身份不是单一特征，而由脸型、五官比例、皮肤纹理、年龄、发型、表情习惯甚至拍摄风格共同构成。模型需要保留这些身份特征，同时允许姿态、表情、风格和场景变化。

<h2 id="q-007">面试问题：Q-Former 注入和 ReferenceNet 注入有什么区别？</h2>

**Q-Former 注入**：

引入一组可学习 query，从身份/视觉特征中读出少量 token：

$$
Z_q = \text{CrossAttn}(Q_{\text{learn}},K_{\text{id}},V_{\text{id}})
$$

优点是 token 化后易接入 Transformer/DiT，适合跨模态对齐；缺点是空间细节可能压缩丢失。

**ReferenceNet 注入**：

直接从参考图提取多尺度空间特征：

$$
F_{\text{ref}}^{(l)} = R^{(l)}(I_{\text{ref}})
$$

并在去噪网络对应层注入。优点是保留纹理和局部细节；缺点是更容易 copy-paste，也更占显存。

**选择建议：**

- 需要灵活文本编辑和轻量接入：Q-Former/ID token 更合适。
- 需要高保真局部细节：ReferenceNet/feature map 更合适。
- 高质量系统常混合使用：全局 ID token + 局部 face reference feature。

<h2 id="q-008">面试问题：如何在 ID 相似度和文本可编辑性之间平衡？</h2>

常见策略：

1. **条件强度调节**：类似 CFG，控制 ID embedding 权重。
2. **延迟注入**：早期让模型确定结构和语义，后期注入 ID 细节。
3. **区域注入**：ID 只作用于脸部区域，不影响背景和服装。
4. **对比对齐损失**：约束有 ID 路径和无 ID 路径在非身份语义上接近。
5. **多任务训练**：混合文本生成、ID 保持、风格编辑任务，避免模型只学重建。
6. **条件 dropout**：随机丢弃 ID 或文本条件，让模型学会不同条件下的鲁棒生成。

**面试金句：**

ID 保持不是把脸复制过去，而是在可编辑生成空间里保持身份不变量。

<h2 id="q-009">面试问题：Copy-Paste 现象如何评估和缓解？</h2>

Copy-Paste 指生成结果过度复刻参考图，而没有根据目标 prompt、姿态或场景产生合理变化。

旧笔记中提到 WithAnyone 的 Copy-Paste 度量思想：比较生成图 $g$ 与目标真实图 $t$、参考图 $r$ 的角距离：

$$
\mathcal{M}_{CP}(g|t,r)=\frac{\theta_{gt}-\theta_{gr}}{\max(\theta_{tr},\epsilon)}
$$

若生成图更接近参考图而不是目标变化，说明复制倾向强。

缓解方法：

- 使用同 ID 多姿态、多表情数据。
- 使用 cross-paired 数据，避免输入输出来自同一图。
- 对参考图做增强，削弱非身份细节。
- 对 ID 和姿态/表情做解耦。
- 训练中加入文本可控性和目标变化损失。

<h2 id="q-010">面试问题：多人 ID 数据集如何构建？</h2>

可参考 MultiID 类数据构建思路：

1. 收集大量单人图像。
2. 用人脸识别 embedding 聚类，形成每个 ID 的参考图像库。
3. 搜索/筛选多人合影或多人场景图。
4. 将多人图中的人脸与单人 ID 库匹配。
5. 构建 `(参考 ID 图, 多人目标图, 人脸位置/ID 标签)` 数据。
6. 做质量过滤：清晰度、美学、遮挡、ID 置信度、年龄跨度、重复图。

**核心难点：**

多人 ID 数据不仅要知道“有哪些人”，还要知道“每张脸对应哪个 ID”，否则多 ID 生成会混脸。

<h1 id="q-011">3. 换脸任务的核心挑战是什么？</h1>

换脸要求把源身份放到目标图像/视频中，同时保留目标属性。核心是身份和属性解耦：

$$
I_{\text{swap}} = G(c_{\text{id}}^{src}, c_{\text{attr}}^{tgt})
$$

其中 $c_{\text{attr}}^{tgt}$ 包括姿态、表情、光照、背景、遮挡、图像质量。

<h2 id="q-012">面试问题：SimSwap、HiFiFace、Face-Adapter、ReFace 的思路有什么区别？</h2>

**SimSwap**：GAN 框架，通过 ArcFace 提取源身份，用身份注入模块将 ID 注入目标特征，并用弱特征匹配损失保留目标属性。

**HiFiFace**：强调高保真换脸和 3D 形状感知，利用更细粒度的人脸先验提升身份和细节。

**Face-Adapter**：更偏 adapter 化思路，把人脸身份作为额外条件注入扩散或生成模型，提升可插拔性。

**ReFace**：强调统一扩散式人脸交换和属性/身份解耦，使用 CLIP/掩码/修复训练等思想，让换脸更鲁棒。

**对比：**

- GAN 换脸速度快，适合实时。
- Diffusion 换脸质量和可控性强，但推理慢。
- 3D/landmark 先验能提升大姿态和表情稳定性。

<h2 id="q-013">面试问题：视频换脸为什么比图像换脸难？</h2>

视频换脸额外需要：

- 每帧身份一致。
- 表情连续。
- 边界不闪烁。
- 光照和颜色稳定。
- 遮挡物不穿帮。
- 大姿态下脸部不变形。

如果逐帧换脸，即使每帧看起来不错，连起来也可能出现 ID 抖动、脸型跳变、颜色闪烁。

解决方式：

- temporal attention / 3D convolution
- 光流一致性损失
- 视频判别器
- 人脸 tracking 和稳定裁剪
- 时序 face embedding 约束
- 后处理去闪烁和颜色匹配

<h2 id="q-014">面试问题：HiFiVFS、VividFace 如何提升视频换脸的一致性和细节？</h2>

**HiFiVFS** 关注高保真视频换脸，基于 SVD 类视频模型，引入：

- 细粒度属性学习：从目标视频提取属性特征。
- 详细身份学习：不只用全局 512 维 ID，而是使用更细的空间身份特征。
- token 化身份细节：把 detailed identity feature 转成 token，注入 U-Net 的 cross-attention 和 temporal attention。

**VividFace** 关注视频人脸交换中的时间一致性、身份保持和复杂遮挡：

- 图像-视频混合训练。
- 特别设计的视频 VAE。
- AIDT 三元组数据，用于身份和属性解耦。
- 遮挡增强，提高真实场景鲁棒性。

**面试总结：**

视频换脸的提升方向是：更细 ID 特征、更强时序建模、更好的属性解耦、更真实的遮挡数据。

<h2 id="q-015">面试问题：换脸如何结合 3DMM 和背景处理？</h2>

3DMM 可提供：

- 脸型和表情参数
- 头姿
- 法线/深度
- UV 纹理
- 可控人脸 mask

它能帮助换脸处理大姿态、表情迁移和光照估计。

当源脸型和目标脸型差异大时，背景处理也很关键。旧笔记提到可构造融合 mask：

$$
M_{\text{swap}}=(M_{\text{tgt}}\cup M_{\text{normal}})\cap(1-M_{\text{occ}})
$$

其中 $M_{\text{occ}}$ 表示头发、手、眼镜等遮挡区域。目标是让源身份形状与目标位置对齐，同时保留背景和遮挡关系。

<h1 id="q-016">4. 虚拟换衣为什么是数字人商业化重要方向？</h1>

虚拟换衣直接对应电商、广告、模特试穿、直播带货和个性化购物，是数字人最容易商业化的方向之一。它能将商品图转成模特图/视频，降低拍摄成本，提升素材生产效率。

<h2 id="q-017">面试问题：视频换衣和图像换衣的核心差异是什么？</h2>

图像换衣只要求单帧合理；视频换衣要求动态合理。

核心差异：

| 维度 | 图像换衣 | 视频换衣 |
| --- | --- | --- |
| 一致性 | 单帧空间一致 | 多帧时序一致 |
| 服装动态 | 不需要 | 要表达褶皱、摆动、遮挡 |
| 纹理 | 单帧清晰即可 | 多帧不能闪烁 |
| 数据 | 图像三元组 | 视频三元组/多帧跟踪 |
| 成本 | 较低 | 更高 |

**面试重点：**

视频换衣不是逐帧图像换衣，必须显式处理服装随动作变化的时序一致性。

<h2 id="q-018">面试问题：Mask-based、Mask-free、蒸馏式虚拟换衣有什么区别？</h2>

**Mask-based**：

把换衣区域视为 inpainting mask：

$$
I_{\text{out}} = G(I_{\text{person}}\odot(1-M),I_{\text{cloth}},M)
$$

优点是区域明确；缺点是依赖 parsing/mask 质量。

**Mask-free**：

不输入二值 mask，通过服装和人体特征对齐让模型自动决定修改区域。优点是保留原图内容更好；缺点是训练和对齐更难。

**蒸馏式方法**：

先用 mask-based teacher 生成结果，再训练 mask-free student。问题是 teacher 的错误可能被学生继承。

**工程折中：**

线上常使用粗 mask + 服装 encoder + pose/densepose + 局部修复的混合方案。

<h2 id="q-019">面试问题：如何处理 mask 引入的形状泄露问题？</h2>

形状泄露指 mask 把原衣服的轮廓暴露给模型，导致模型推理时倾向填满原衣服形状，而不是生成新衣服应有形状。

FitDiT 类思路中，可使用扩张-放松 mask：

- 对上衣：包含手部关键点和上衣解析区域。
- 对下装：结合足部关键点和下装解析。
- 对连衣裙：同时考虑手部、足部和裙装区域。
- 对 mask 边界随机扩张，覆盖部分不变区域，减少原衣形状泄露。

**关键原则：**

mask 要足够大，避免泄露原衣轮廓；但不能过大，否则会破坏人脸、手、头发、背景等应保留内容。

<h2 id="q-020">面试问题：如何构造成品模特试穿视频生成链路？</h2>

一套商品试穿视频链路：

1. **商品解析**：输入商品图或 item id，提取品类、颜色、材质、版型、季节、场景。
2. **标签匹配**：用 VLM/规则生成结构化标签，如 `风衣-秋冬-商务-长款`。
3. **模板选择**：从动作/镜头模板库中选择匹配服装的模特视频。
4. **逻辑校验**：例如无口袋衣服不匹配插兜动作，裙装不匹配不合理腿部动作。
5. **虚拟换衣生成**：服装图 + 模特视频 + 姿态/解析条件。
6. **多模态增强**：生成文案、TTS、配乐、字幕和镜头剪辑。
7. **质检**：纹理清晰、服装类别正确、无闪烁、无遮挡错误、品牌/logo 不变形。

<h1 id="q-021">5. ID 保持、换脸和换衣如何做数据与评估？</h1>

这些任务的数据质量非常关键，因为模型需要学习身份、服装、姿态、表情、背景之间的解耦。数据构造不好会直接导致 copy-paste、身份漂移、服装变形和时序闪烁。

<h2 id="q-022">面试问题：虚拟换衣如何构造和筛选三元组数据？</h2>

虚拟换衣三元组：

$$
(I_{\text{person}}, I_{\text{cloth}}, I_{\text{target}})
$$

构造方式：

1. 从电商数据中收集商品图和模特上身图。
2. 使用人体 parsing 得到衣服区域。
3. 构造 agnostic person representation，移除原衣区域。
4. 使用图像编辑模型或 inpainting 模型扩充数据。
5. 保留商品图、人物图、目标上身图之间的 SKU 对应关系。

筛选指标：

- 姿态一致性。
- 衣服纹理一致性。
- 人脸未被误修复。
- 遮挡合理。
- 图像清晰和美学分。
- 人工抽检。

<h2 id="q-023">面试问题：虚拟换衣常用数据集有哪些？</h2>

旧笔记中提到的数据集：

| 数据集 | 特点 |
| --- | --- |
| VITON-HD | 上衣试穿，高分辨率，白底，常用基准 |
| FashionTryOn | 三元组图像数据，包含不同姿态 |
| StreetTryOn | 街景背景，更接近真实场景 |
| DressCode | 品类更丰富，包含上衣、下装、裙子 |

**面试要点：**

白底数据容易训练但泛化到真实街景差；街景数据更真实但遮挡、光照、姿态复杂。视频换衣还需要视频级数据，难度更高。

<h2 id="q-024">面试问题：如何通过文本控制衣服尺寸、宽松度和袖长？</h2>

关键是把口语描述转成结构化服装属性 token：

- `oversized fit`
- `slim fit`
- `longline hem`
- `cropped length`
- `dropped shoulders`
- `long sleeves`
- `rolled-up sleeves`

可用 VLM/LLM 把用户输入重写：

```text
把这件 T 恤改成宽松一点，下摆盖到臀部
```

转为：

```text
upper body, white T-shirt, oversized fit, longline hem covering hips, loose sleeves
```

然后通过 cross-attention 或 prompt-aware mask 让这些 token 只作用于衣服区域。

评估可看文本-区域响应，例如旧笔记中的 SRR/SRF 思路：统计衣服区域对尺寸 token 的 attention 响应是否足够强、是否集中在正确区域。

<h2 id="q-025">面试问题：线上发现纹理糊、头发遮挡、服饰动作冲突如何处理？</h2>

**纹理糊：**

- 服装图先超分。
- 模型多尺度训练。
- 局部 patch loss / perceptual loss。
- 高分辨率二阶段重绘。

**头发遮挡：**

- 单独做人发 segmentation。
- 换衣时保护头发 mask。
- 最后把头发层 alpha blend 回来。
- 边缘羽化或 Poisson blending。

**服饰动作冲突：**

- 建立服饰标签和动作模板规则。
- 用 VLM 判断服装是否有口袋、裙摆、袖长等。
- 过滤不合理模板，例如无口袋服装不要匹配插兜动作。
- 对复杂服装使用更保守的动作库。

**一句话：**

虚拟换衣上线不是只靠模型，商品理解、动作模板、分割层级、后处理和质检规则同样重要。
