<h1 id="q-001">1. NLP 经典模型如何演进到大语言模型？</h1>

NLP 经典模型的演进主线可以概括为：

```text
离散符号 -> 分布式词向量 -> 序列模型 -> Attention -> Transformer -> 预训练语言模型 -> 指令对齐与多模态大模型
```

本节覆盖 NLP 发展史、词向量、Seq2Seq、Attention、机器翻译、情感分析、问答系统、知识图谱、强化学习和元学习等跨周期核心，并补充 BERT、GPT、T5、CLIP、BLIP、Flamingo、LLaVA、Florence-2 等 AIGC 时代模型。

<h2 id="q-002">面试问题：词向量 Word2Vec / GloVe 解决了什么问题？</h2>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

词向量解决的是离散词无法直接表达语义相似性的问题。

One-hot 表示的问题：

- 维度很高。
- 稀疏。
- 不表达词与词之间的语义关系。
- “猫”和“狗”的距离与“猫”和“汽车”的距离没有区别。

Word2Vec 将词映射到低维稠密向量，使相似词在向量空间中更接近。

常见训练方式：

- CBOW：根据上下文预测中心词。
- Skip-gram：根据中心词预测上下文。

GloVe 则利用全局词共现统计学习词向量。

大模型时代的联系：

- LLM 的 token embedding 是词向量思想的延续。
- CLIP 的图文 embedding 是跨模态词向量思想的扩展。
- RAG 中的 embedding 检索也依赖“语义相似映射到向量相似”。

<h2 id="q-003">面试问题：RNN、LSTM、GRU 为什么曾经是序列建模主力？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

RNN 通过隐藏状态递归处理序列：

$$
h_t = f(x_t, h_{t-1})
$$

它适合处理文本、语音、时间序列等有顺序的数据。

RNN 的问题：

- 长序列梯度消失或爆炸。
- 难以捕获长距离依赖。
- 训练难并行。

LSTM 和 GRU 通过门控机制缓解长依赖问题。

LSTM 主要门：

- 输入门。
- 遗忘门。
- 输出门。

GRU 更简洁，使用更新门和重置门。

为什么现在仍要理解：

- 语音、时间序列、控制系统中仍有应用。
- LSTM 的门控思想影响了后续模型设计。
- 面试中常用来对比 Transformer 的并行性和长依赖建模能力。

<h2 id="q-004">面试问题：Seq2Seq 和 Attention 如何推动机器翻译？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

Seq2Seq 使用 encoder 将输入序列编码成上下文向量，再由 decoder 生成输出序列。

问题：

- 如果只用一个固定长度向量承载整个输入，长句信息容易丢失。

Attention 的作用：

- decoder 每一步生成时，可以动态关注 encoder 的不同位置。
- 不再只依赖最后一个固定向量。
- 提供对齐能力，特别适合机器翻译。

通俗理解：

翻译一个英文句子到中文时，生成每个中文词并不需要同等关注所有英文词，而是应该关注当前最相关的源词。Attention 就是学习这种动态对齐。

这条路线最终通向 Transformer：如果 attention 足够强，序列建模可以不再依赖 RNN。

<h1 id="q-005">2. Transformer 为什么成为 NLP 和多模态大模型的基础结构？</h1>

Transformer 用 self-attention 替代循环结构，解决了 RNN 难并行和长距离依赖建模不足的问题。

优势：

- 并行训练。
- 全局 token 交互。
- 易于规模化。
- 适配文本、图像、音频、视频、多模态 token。
- 支撑 BERT、GPT、T5、ViT、DiT、多模态 LLM 等模型。

<h2 id="q-006">面试问题：Self-Attention 的核心计算是什么？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

Self-Attention 的核心是 Query、Key、Value。

给定输入 $X$：

$$
Q = XW_Q,\quad K = XW_K,\quad V = XW_V
$$

Attention 输出：

$$
Attention(Q,K,V)=softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

含义：

- Query 表示当前位置想找什么信息。
- Key 表示每个位置能提供什么索引。
- Value 表示每个位置实际提供的信息。

为什么除以 $\sqrt{d_k}$：

- 避免点积过大导致 softmax 饱和。
- 稳定梯度。

多头注意力：

- 不同 head 学习不同关系。
- 有的关注语法，有的关注实体，有的关注长距离依赖。

AIGC 联系：文本大模型、ViT、DiT、视频 Transformer、多模态 cross-attention 都建立在这个公式之上。

<h2 id="q-007">面试问题：BERT 和 GPT 的预训练目标有什么区别？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

BERT 和 GPT 都是 Transformer 预训练模型，但方向不同。

| 维度 | BERT | GPT |
| --- | --- | --- |
| 结构 | Transformer Encoder | Transformer Decoder |
| 注意力 | 双向上下文 | 因果单向 |
| 预训练目标 | Masked Language Modeling | Next Token Prediction |
| 擅长 | 理解、分类、抽取 | 生成、对话、推理 |
| 典型任务 | NER、分类、阅读理解 | 文本生成、代码生成、Agent |

BERT 通过 mask 部分 token，让模型根据双向上下文预测被遮挡词。

GPT 使用自回归目标：

$$
P(x) = \prod_t P(x_t | x_{<t})
$$

大模型时代 GPT 路线成为主流生成范式，因为 next token prediction 可以自然扩展到长文本、代码、工具调用和多模态 token 生成。

<h2 id="q-008">面试问题：T5 的 Text-to-Text 思想有什么意义？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

T5 将所有 NLP 任务统一为文本到文本。

例子：

- 翻译：`translate English to German: ...`
- 摘要：`summarize: ...`
- 分类：输入文本，输出类别文本。
- 问答：输入上下文和问题，输出答案。

意义：

- 统一输入输出格式。
- 统一训练和推理接口。
- 适合多任务学习。
- 对后来的 instruction tuning 和 prompt-based learning 有启发。

AIGC 联系：

今天的 LLM 基本都把任务统一为“输入指令，输出文本 / 结构化文本 / 工具调用”，这与 T5 的统一任务表述思想一脉相承。

<h1 id="q-009">3. 传统 NLP 任务在大模型时代还重要吗？</h1>

重要。大模型不是取消传统 NLP 任务，而是把很多任务统一到一个生成或理解框架中。

传统任务仍然有价值：

- 用于评估模型能力。
- 用于构建业务系统。
- 用于理解 LLM 输出。
- 用于 RAG、Agent、搜索、推荐、知识库等系统。

<h2 id="q-010">面试问题：机器翻译、情感分析、问答系统的经典路线是什么？</h2>

**难度评分：⭐⭐ (2/5)  |  考察频率：⭐⭐⭐ (3/5)**

机器翻译：

- 统计机器翻译。
- Seq2Seq。
- Seq2Seq + Attention。
- Transformer。
- 多语言大模型。

情感分析：

- 词典规则。
- 传统机器学习 + 特征工程。
- CNN/RNN 文本分类。
- BERT fine-tuning。
- LLM prompt / instruction。

问答系统：

- 检索式 QA。
- 阅读理解式 QA。
- 生成式 QA。
- RAG + LLM。
- Agentic QA，结合工具、记忆和检索。

面试表达：传统任务的名字没变，但底层模型从任务专用模型变成了预训练模型和通用大模型。

<h2 id="q-011">面试问题：知识图谱和 RAG 有什么关系？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

知识图谱用实体和关系表示结构化知识。

RAG 用检索增强生成，让 LLM 在生成前获取外部知识。

两者关系：

- 知识图谱可作为 RAG 的结构化知识源。
- 图谱能提供实体关系、路径推理和约束。
- RAG 能弥补纯知识图谱表达不自然、覆盖不足的问题。
- Graph RAG 将文档 chunk、实体、关系、社区结构结合起来。

对比：

| 维度 | 知识图谱 | RAG |
| --- | --- | --- |
| 知识形态 | 结构化实体关系 | 文档、向量、图、数据库 |
| 优势 | 可解释、关系清晰 | 覆盖广、生成自然 |
| 难点 | 构建和维护成本高 | 检索质量和幻觉控制 |
| 结合方式 | 实体链接、图检索、路径证据 | LLM 生成答案 |

<h2 id="q-012">面试问题：NLP 与强化学习、元学习、Agent 有什么联系？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐ (3/5)**

NLP 与强化学习：

- 对话系统可以看作序列决策。
- RLHF 用强化学习根据人类偏好优化语言模型。
- Agent 工具调用可视为策略选择。

NLP 与元学习：

- 少样本学习、prompt learning、in-context learning 都与快速适配有关。
- 大模型通过上下文示例完成任务，体现了某种元学习能力。

NLP 与 Agent：

- LLM 负责理解指令、规划步骤、生成工具调用。
- RAG 提供知识。
- 外部工具执行动作。
- 记忆系统维护长期状态。

从经典 NLP 到 Agent 的转变是：模型不再只是处理文本，而是通过文本接口组织工具、知识和行动。

<h1 id="q-013">4. 多模态模型如何从 CLIP 走向视觉语言助手？</h1>

多模态模型的发展大致可以分为：

```text
图文对齐 -> 图文理解/生成 -> 冻结 LLM 桥接视觉 -> 多模态指令微调 -> 统一视觉任务模型
```

代表模型：

- CLIP
- BLIP / BLIP-2
- Flamingo
- LLaVA
- Florence-2

<h2 id="q-014">面试问题：CLIP 的图文对比学习为什么重要？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

CLIP 使用大规模图像-文本对进行对比学习。

训练目标：

- 匹配的图像和文本相似度更高。
- 不匹配的图像和文本相似度更低。

重要性：

- 支持 zero-shot 分类。
- 支持图文检索。
- 为开放词汇检测和分割提供语义基础。
- 为文生图模型提供文本语义条件。
- 让视觉模型从固定类别分类走向开放词汇理解。

典型用法：

```text
图像 -> image encoder -> image embedding
文本 -> text encoder -> text embedding
相似度比较 -> 分类 / 检索 / 对齐
```

<h2 id="q-015">面试问题：BLIP、BLIP-2、Flamingo 分别解决了什么问题？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

BLIP：

- 统一视觉语言理解和生成。
- 使用 captioning、image-text matching、contrastive learning 等任务。
- 关注高质量图文数据自举。

BLIP-2：

- 冻结图像编码器和大语言模型。
- 使用轻量 Q-Former 连接视觉特征和语言模型。
- 降低多模态预训练成本。

Flamingo：

- 面向 few-shot 多模态学习。
- 能处理交错的图像、视频和文本输入。
- 通过 Perceiver Resampler 和 gated cross-attention 将视觉信息接入语言模型。

对比：

| 模型 | 核心点 | 启发 |
| --- | --- | --- |
| BLIP | 图文理解和生成统一 | 数据自举和多任务预训练 |
| BLIP-2 | 冻结大模型，轻量桥接 | 高效多模态适配 |
| Flamingo | few-shot 多模态上下文学习 | 多模态 in-context learning |

<h2 id="q-016">面试问题：LLaVA 的视觉指令微调有什么意义？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐⭐ (5/5)**

LLaVA，Large Language and Vision Assistant，将视觉编码器和 LLM 连接起来，并使用多模态指令数据进行训练。

核心思想：

- 使用 CLIP 等视觉编码器提取图像特征。
- 用 projection layer 将视觉特征对齐到 LLM embedding 空间。
- 使用 GPT-4 生成的图文指令数据做 visual instruction tuning。
- 让模型具备图像问答、多模态对话和视觉推理能力。

意义：

- 从固定任务 VLM 走向通用视觉语言助手。
- 证明指令微调同样适用于多模态。
- 成为开源多模态 LLM 的重要路线。

面试中可说：CLIP 解决“图文对齐”，LLaVA 进一步解决“视觉信息如何接入 LLM 并按指令对话”。

<h2 id="q-017">面试问题：Florence-2 为什么被称为统一视觉任务模型？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐ (3/5)**

Florence-2 的目标是用统一表示处理多种视觉任务，包括：

- 图像描述。
- 目标检测。
- OCR。
- Phrase grounding。
- 分割相关任务。

核心思想：

- 将多种视觉任务统一为 prompt-based sequence-to-sequence 问题。
- 使用大规模 FLD-5B 数据构建统一监督。
- 输出可以是文本、框、区域、结构化结果。

为什么重要：

- 传统视觉任务往往每个任务一个专用头。
- Florence-2 尝试用一个模型统一多任务表示和输出。
- 这和大模型时代“统一接口、统一任务格式”的趋势一致。

<h1 id="q-018">5. NLP、多模态和生成模型如何组合成 AIGC 系统？</h1>

真实 AIGC 系统通常是多模型组合：

- LLM 负责语言理解、规划和生成。
- 多模态模型负责图像/视频/文档理解。
- 检索系统提供外部知识。
- 生成模型负责图像、视频、音频或 3D 生成。
- 检测、分割、OCR 等模型提供结构化视觉信息。

<h2 id="q-019">面试问题：文本编码器在文生图、视频生成和多模态理解中有什么作用？</h2>

**难度评分：⭐⭐⭐ (3/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

文本编码器负责将 prompt 转换为语义条件。

在文生图中：

- 文本编码器输出 token embedding。
- 扩散模型通过 cross-attention 或 conditioning 接收文本条件。
- prompt 质量、tokenization、文本编码器能力会影响生成结果。

在视频生成中：

- 文本条件控制主体、动作、风格和场景。
- 还需要时序一致性模块处理跨帧关系。

在多模态理解中：

- 文本和图像被映射到共同空间或通过 cross-attention 融合。
- LLM 根据视觉 token 和文本指令生成答案。

面试要点：文本编码器不是简单“把字变数字”，而是提供生成模型理解用户意图的语义桥梁。

<h2 id="q-020">面试问题：RAG、多模态 Agent 和工具调用如何复用经典 NLP 模型思想？</h2>

**难度评分：⭐⭐⭐⭐ (4/5)  |  考察频率：⭐⭐⭐⭐ (4/5)**

RAG 复用的 NLP 思想：

- 信息检索。
- 语义向量。
- 问答系统。
- 摘要和生成。
- 知识图谱和实体关系。

多模态 Agent 复用的模型思想：

- Attention：对不同模态和工具结果动态加权。
- Seq2Seq：从任务输入生成行动序列。
- RL：根据反馈优化策略。
- 迁移学习：把预训练模型适配到具体工具和业务。

工具调用：

- 函数名、参数、返回值可以看作结构化语言。
- LLM 生成工具调用类似生成程序或动作序列。
- 检索、OCR、检测、分割、代码执行都可以成为工具。

总结：AIGC Agent 不是脱离经典 NLP 的新物种，而是把语言建模、检索、问答、规划和强化学习组合成可行动系统。

## 高频速记

- Word2Vec 解决离散词语义表示问题。
- Seq2Seq 用 Encoder-Decoder 处理序列到序列。
- Attention 解决固定上下文瓶颈，提供动态对齐。
- Transformer 用 Self-Attention 支撑并行和长距离建模。
- BERT 偏理解，GPT 偏生成，T5 统一 text-to-text。
- CLIP 用图文对比学习打开开放词汇视觉理解。
- BLIP/BLIP-2 连接视觉语言理解和生成。
- Flamingo 强调 few-shot 多模态上下文学习。
- LLaVA 强调视觉指令微调。
- Florence-2 尝试统一多种视觉任务表示。
